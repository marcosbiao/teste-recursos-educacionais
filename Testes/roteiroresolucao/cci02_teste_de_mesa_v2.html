CCI02

Verificando o caminho da entrada até a saída

Roteiro de resolução para acompanhar manualmente entrada, processamento, mudanças de estado e saída final.

Tente resolver cada etapa antes de usar as ajudas e antes de ver a orientação esperada.

Problema a resolver

Executar manualmente um programa simples para verificar se a saída obtida corresponde à esperada.

O teste de mesa acompanha cada linha do programa e registra como os valores das variáveis mudam até chegar à saída.

Se eu usar estes valores de entrada, qual será a saída passo a passo?

Use o programa de referência abaixo apenas como base para o teste de mesa. O foco é rastrear os valores, não escrever um novo código.

int numero;
int dobro;

scanf("%d", &numero);
dobro = numero * 2;
printf("Dobro: %d", dobro);

Roteiro de resolução

1
Etapa 1 Habilidade: Rastrear execução

Escolher valores de teste

Quais valores de entrada serão usados para acompanhar o programa manualmente?

Defina um caso de teste mínimo para executar o programa.

Precisa de ajuda?

Pista: escolha um valor simples para facilitar o acompanhamento manual.
Pergunta guia: qual variável recebe valor diretamente pelo comando de leitura?
Exemplo parecido: se um programa calcula triplo, usar entrada 4 facilita verificar se o resultado será 12.
Começo da resposta: variável de entrada = ____; valor de entrada = ____; resultado esperado = ____.

Orientação esperada: sua resposta deve indicar que a entrada é atribuída à variável numero. Para um valor simples, como 7, o resultado esperado é 14, pois a expressão de processamento calcula o dobro da entrada.

Pista: o comando scanf altera apenas a variável indicada nele.

Comparar saída esperada e obtida

A saída obtida no teste de mesa corresponde ao que era esperado?

Registre a saída e indique se há inconsistência.

Precisa de ajuda?

Pista: validar não é apenas calcular, é comparar o que deveria sair com o que saiu no rastreamento.
Pergunta guia: o valor numérico exibido está igual ao valor esperado?
Exemplo parecido: se o esperado era 8.0 e o teste mostrou 7.0, é preciso revisar o processamento ou os valores usados.
Começo da resposta: esperado = ____; obtido = ____; comparação = ____; conclusão = ____.

Orientação esperada: sua resposta deve comparar a saída prevista com a saída registrada no teste de mesa. Se o valor escolhido foi 7, o valor numérico coerente para a saída é 14. Caso a saída obtida seja diferente, revise se a leitura, a atribuição ou a saída formatada foram acompanhadas corretamente.

Erros comuns tratados

Pular linhas do algoritmo

Por que acontece: o aluno tenta ir direto para o resultado sem acompanhar a execução.

Como evitar: registrar cada linha na ordem em que aparece.

Atualizar variável no momento errado

Por que acontece: o aluno altera uma variável antes da linha que realmente atribui valor a ela.

Como evitar: mudar o estado apenas quando houver leitura ou atribuição.

Não comparar com o esperado

Por que acontece: o aluno calcula a saída, mas não verifica se ela corresponde ao objetivo.

Como evitar: escrever saída esperada, saída obtida e conclusão da comparação.

Fechamento pedagógico

Ao concluir este roteiro, o aluno deve conseguir executar manualmente um programa simples, acompanhar mudanças de estado e validar se a saída obtida corresponde à saída esperada.
