CCI03

Escolhendo tipos adequados para os dados

Roteiro de resolução para analisar dados, escolher tipos em C e evitar perda de informação em expressões.

Tente resolver cada etapa antes de usar as ajudas e antes de ver a orientação esperada.

Problema a resolver

Definir tipos de dados para um problema envolvendo idade, preço, quantidade e média.

O programa precisa representar dados simples de forma coerente para realizar operações aritméticas sem perder informação importante.

Que tipo de dado representa melhor cada informação do problema?

Antes de avançar, pense se cada dado representa contagem, valor com casas decimais, texto curto ou resultado de uma operação.

Dados do problema: idade, preço, quantidade e média.

Foco: escolher tipos adequados e justificar a decisão a partir do uso de cada dado.

Pista: antes de escolher o tipo em C, identifique o que o dado representa no mundo do problema.
Pergunta guia: o dado representa uma contagem, um valor com centavos, um resultado de divisão ou um único caractere, como A, B ou C?

Precisa de ajuda?

Pista: em C, int é usado para inteiros, float para números reais e char para um caractere.
Pergunta guia: o valor pode precisar de casas decimais? Se sim, int pode não preservar a informação.
Exemplo parecido: quantidade de livros pode ser int, preço de um livro costuma ser float e uma categoria única, como A, pode ser char.
Começo da resposta: idade: int. quantidade: int. preço: ____. média: ____. categoria: ____.

Orientação esperada: uma escolha coerente é declarar idade e quantidade como int, pois representam contagens inteiras. preco e media devem ser representados como float, pois podem envolver casas decimais. Para um exemplo auxiliar de código de categoria com apenas um símbolo, como A, o tipo char é adequado.

Precisa de ajuda?

Prever risco de perda de informação

Que problema pode ocorrer se o tipo escolhido não preservar o valor necessário?

Identifique o risco em cada situação e proponha uma correção.

Situação Risco provável Correção
int preco = 19.90;

Precisa de ajuda?

Pista: perda de informação ocorre quando o tipo não consegue representar parte do valor.
Pergunta guia: se um valor tem casas decimais e é guardado como int, o que pode acontecer?
Exemplo parecido: guardar 8.7 em uma variável int pode descartar a parte decimal.
Começo da resposta: preço e média devem evitar perda de ____. quantidade pode continuar como ____.

Orientação esperada: revise se você reconheceu risco de truncamento ou perda da parte decimal em preço e média quando são declarados como int. Para esses casos, float é mais adequado. Para quantidade inteira, manter int é coerente quando o problema trabalha com unidades completas. int idade; int quantidade; float preco; float media;

Evidências esperadas

Erros comuns tratados

Usar inteiro para valores reais

Por que acontece: o aluno pensa apenas no número, sem observar se há casas decimais.

Como evitar: usar float quando o dado puder ter parte decimal.

Misturar tipos sem necessidade

Por que acontece: o aluno escolhe tipos diferentes sem relacionar a escolha ao uso do dado.

Como evitar: decidir o tipo a partir da representação necessária e da operação realizada.

Declarar variável sem uso claro

Por que acontece: a variável é criada antes de entender sua função no problema.

Como evitar: declarar apenas dados que serão lidos, calculados ou exibidos.
