Precisa de ajuda?

Pista: uma função que calcula e devolve um inteiro deve ter tipo de retorno int.
Pergunta guia: o procedimento precisa devolver algum valor ou apenas imprimir?
Exemplo parecido: int calcularMedia(...) retorna um valor. void mostrarMedia(...) apenas exibe.
Começo da resposta: a função começa com int encontrarMaior(...). O procedimento começa com void mostrarMaior(...).

Orientação esperada: a função precisa retornar um inteiro, então sua assinatura usa int. O procedimento só apresenta o resultado, então usa void. Compare sua tentativa com o código final: 
#include <stdio.h>

int encontrarMaior(int numero1, int numero2, int numero3) {
    int maiorValor = numero1;

    if (numero2 > maiorValor) {
        maiorValor = numero2;
    }

    if (numero3 > maiorValor) {
        maiorValor = numero3;
    }

    return maiorValor;
}

void mostrarMaior(int maiorValor) {
    printf("Maior valor: %d\n", maiorValor);
}

int main() {
    int numero1;
    int numero2;
    int numero3;
    int maiorValor;

    printf("Digite o primeiro numero: ");
    scanf("%d", &numero1);

    printf("Digite o segundo numero: ");
    scanf("%d", &numero2);

    printf("Digite o terceiro numero: ");
    scanf("%d", &numero3);

    maiorValor = encontrarMaior(numero1, numero2, numero3);
    mostrarMaior(maiorValor);

    return 0;
}

Erros comuns tratados

Ler dados dentro da função

Por que acontece: o aluno mistura entrada de dados com cálculo.

Como evitar: manter a leitura no main e passar os valores como parâmetros.

Usar procedimento para calcular

Por que acontece: o aluno não diferencia função com retorno de procedimento sem retorno.

Como evitar: usar função para determinar o maior valor e procedimento apenas para mostrar.

Perder o maior já encontrado

Por que acontece: o aluno compara valores diretamente sem guardar o maior atual.

Como evitar: iniciar maiorValor com numero1 e atualizar somente quando outro número for maior.
