CCI07 Erro de interface

Parâmetros insuficientes ou desnecessários

Painel de diagnóstico para reconhecer quando uma função depende de variáveis externas ou recebe dados que não usa.

Nível de gravidade
Médio

O programa pode até executar, mas a função fica frágil, difícil de testar e pouco reutilizável.

Sintoma observado

A função usa dados que não recebeu

A função depende de variáveis externas ou declara parâmetros que não participam do cálculo.

Impacto no programa

Interface do módulo fica inconsistente

Como a entrada da função não representa os dados realmente usados, o módulo passa a depender do contexto em que foi chamado.

Enunciado analisado

Referência para comparar pedido e código

O enunciado abaixo será usado como referência diagnóstica. A análise deve verificar se a função recebe explicitamente os dados necessários para calcular a média.

Calcule a média de duas notas informadas no programa principal. Use uma função para calcular a média e mostre o resultado final.

Primeira análise

Código com erro para diagnóstico inicial

Antes de ver a explicação, compare o enunciado com o código e observe se a função consegue trabalhar apenas com os dados recebidos em sua interface.

Código com erro 
#include <stdio.h>

float nota1, nota2;

float calcularMedia() {
    // A função usa dados que não recebeu por parâmetro.
    return (nota1 + nota2) / 2.0;
}

int main() {
    float media;

    scanf("%f", &nota1);
    scanf("%f", &nota2);

    media = calcularMedia();
    printf("Media: %.2f", media);

    return 0;
}

Causa provável

O aluno separa a função, mas não define sua entrada

Esse erro aparece quando a modularização é feita antes de decidir quais informações a função precisa receber para trabalhar sozinha. O aluno cria a função, mas deixa os dados escondidos em variáveis globais.

Evidência no código

Sinais observáveis

  • Padrão detectável: a função calcula a média, mas não possui parâmetros.
  • Comportamento observado: o resultado depende de variáveis declaradas fora da função.
  • Teste revelador: tentar chamar a função com outras notas sem alterar variáveis globais.
Quando aparece

Durante a definição da interface

Ocorre quando o aluno não lista os dados de entrada da função. A pergunta central é simples: quais valores a função precisa receber para executar sua tarefa?

Exemplo com erro

Código com erro

Código com erro 
#include <stdio.h>

float nota1, nota2;

float calcularMedia() {
    // Problema: a função usa variáveis externas.
    return (nota1 + nota2) / 2.0;
}

int main() {
    float media;

    scanf("%f", &nota1);
    scanf("%f", &nota2);

    media = calcularMedia();
    printf("Media: %.2f", media);

    return 0;
}

O trecho calcula a média, mas a função não recebe as notas como entrada. Ela depende de nota1 e nota2 declaradas fora dela, o que torna a interface da função incompleta.

Correção mínima

Código corrigido

Código corrigido 
#include <stdio.h>

float calcularMedia(float n1, float n2) {
    return (n1 + n2) / 2.0;
}

int main() {
    float nota1, nota2, media;

    scanf("%f", &nota1);
    scanf("%f", &nota2);

    media = calcularMedia(nota1, nota2);
    printf("Media: %.2f", media);

    return 0;
}

A correção passa as duas notas como parâmetros. Assim, a função recebe explicitamente os dados de que precisa e pode ser chamada com outros valores sem depender de variáveis externas.

Antes e depois

Diferença essencial entre o erro e a correção

Lógica incorreta

A função consulta dados que estão fora dela. A chamada não mostra quais valores entram no cálculo.

media = calcularMedia();
calcularMedia usa nota1 e nota2 globais

Lógica correta

A função recebe os dados necessários. A chamada mostra de forma explícita quais valores participam do cálculo.

media = calcularMedia(nota1, nota2);
calcularMedia recebe n1 e n2
Por que isso acontece

Parâmetros definem a entrada da função

Os parâmetros dizem quais dados entram na função. Quando esses dados são recebidos de forma explícita, o módulo fica mais claro, testável e reutilizável.

Como identificar

Estratégia diagnóstica

  • Liste todos os dados usados dentro da função.
  • Verifique se cada dado chega por parâmetro ou é produzido dentro da função.
  • Desconfie de variáveis globais usadas diretamente no cálculo.
Forma correta

Passar apenas os dados necessários

  1. Leia o enunciado e identifique a tarefa da função.
  2. Marque os dados que a função precisa para executar essa tarefa.
  3. Coloque esses dados na lista de parâmetros.
  4. Remova parâmetros que não são usados dentro da função.
  5. Chame a função informando os argumentos correspondentes.
Perguntas de verificação

Verifique se você reconheceu o erro

O que os parâmetros representam em uma função?

Representam os dados que a função recebe para executar sua tarefa. No caso da média, as notas devem chegar como parâmetros.

Qual sinal indica que a função depende de variáveis externas?

Um sinal forte é a função usar uma variável que não foi declarada dentro dela e não aparece na lista de parâmetros.

Em uma função de média de duas notas, quais parâmetros são necessários?

São necessárias as duas notas. A média pode ser calculada e retornada pela própria função, sem ser recebida como parâmetro.

Teste rápido

Analise o código errado e marque o diagnóstico

Mini enunciado: crie uma função que calcule o desconto de um produto a partir do preço e do percentual informados no programa principal. A função deve retornar o valor do desconto.

#include <stdio.h>

float preco, percentual;

float calcularDesconto() {
    return preco * percentual / 100.0;
}

int main() {
    float desconto;

    scanf("%f", &preco);
    scanf("%f", &percentual);

    desconto = calcularDesconto();
    printf("Desconto: %.2f", desconto);

    return 0;
}
