CCI04 Controlar fluxo por decisões condicionais e expressões lógicas Erro lógico condicional

Condição invertida

Este card ajuda a reconhecer, compreender e corrigir uma condição que representa a regra do problema ao contrário.

Gravidade

Alto

O erro pode alterar completamente a decisão tomada pelo programa.

Sintoma observado

O programa executa o bloco errado para determinados casos.

Impacto no programa

A lógica condicional aceita casos que deveriam ser rejeitados e rejeita casos que deveriam ser aceitos.

Enunciado analisado

Use este enunciado como referência diagnóstica para comparar a regra pedida com a regra implementada no código.

Leia a idade de uma pessoa. Se a idade for maior ou igual a 18, mostre “Maior de idade”. Caso contrário, mostre “Menor de idade”.

A regra central é: idade 18 ou maior deve executar o bloco de maioridade.

Causa provável

Esse erro costuma aparecer quando o aluno entende a ideia geral do problema, mas converte a regra para o operador relacional oposto. A troca de maior por menor, igual por diferente ou a leitura apressada do caso limite faz a condição representar o contrário do que o enunciado pede.

Evidência no código

  • Padrão detectável O enunciado pede maior ou igual a 18, mas o código testa idade menor que 18.
  • Comportamento observado Uma idade menor que 18 pode receber a mensagem de maioridade.
  • Teste revelador Testar 17, 18 e 19 mostra se o bloco correto está sendo executado.

Quando aparece

Quando o aluno troca maior por menor, igual por diferente ou confunde a regra do problema. Em condições, uma pequena inversão no operador muda o conjunto de casos que entra no bloco do if.

Exemplo com erro

Código com erro 
#include <stdio.h>

int main() {
    int idade;

    printf("Digite a idade: ");
    scanf("%d", &idade);

    if (idade < 18) { // erro: condição invertida
        printf("Maior de idade");
    } else {
        printf("Menor de idade");
    }

    return 0;
}

O enunciado pede que idade maior ou igual a 18 seja classificada como maioridade. No código, a condição idade < 18 direciona menores de idade para o bloco “Maior de idade”. O caso limite 18 também fica errado, pois cai no else.

Correção mínima

Código corrigido 
#include <stdio.h>

int main() {
    int idade;

    printf("Digite a idade: ");
    scanf("%d", &idade);

    if (idade >= 18) {
        printf("Maior de idade");
    } else {
        printf("Menor de idade");
    }

    return 0;
}

A correção muda apenas a condição do if. Agora o bloco “Maior de idade” é executado quando idade é 18 ou maior, exatamente como o enunciado determina.

Entrada, processamento e saída

Parte do problema O que o enunciado pede O que o código com erro fez Diagnóstico
Entrada Ler uma idade inteira. Lê uma idade inteira. A entrada está adequada. O problema não está na leitura do dado.
Processamento Testar se idade é maior ou igual a 18. Testa se idade é menor que 18. A condição representa a regra no sentido contrário.
Saída Mostrar maioridade para idade 18 ou maior. Mostra maioridade para idade menor que 18. A saída fica incorreta porque depende da condição invertida.

Antes e depois

Lógica incorreta

se idade < 18, então mostrar "Maior de idade"

A condição seleciona menores de idade, mas o bloco imprime a mensagem de maioridade.

Lógica correta

se idade >= 18, então mostrar "Maior de idade"

A condição seleciona exatamente os casos que devem receber a mensagem de maioridade.

Por que isso acontece

A condição precisa representar exatamente a regra de decisão do problema. Se a regra diz “maior ou igual a 18”, o código não pode testar “menor que 18” para executar o mesmo bloco.

Como identificar

Teste a condição com valores de exemplo, incluindo casos limites.

Idade Saída esperada
17 Menor de idade
18 Maior de idade
19 Maior de idade

Forma correta

Reescreva a condição em linguagem natural e depois converta para código.

  1. Leia a regra de decisão.
  2. Escreva o caso que deve entrar no if.
  3. Marque o caso limite.
  4. Escolha o operador relacional adequado.
  5. Teste exemplos antes de finalizar.

Perguntas de verificação

Se o enunciado pede idade maior ou igual a 18, qual condição representa essa regra?

A condição adequada é idade >= 18. Ela inclui o caso limite 18 e todos os valores maiores.

Por que idade < 18 está incorreta para imprimir “Maior de idade”?

Porque idade < 18 seleciona menores de idade. Se esse teste controla o bloco “Maior de idade”, a decisão fica invertida.

Qual valor deve ser testado como caso limite nesse problema?

O valor 18. Ele separa os dois grupos e revela se a condição incluiu corretamente a idade mínima para maioridade.
