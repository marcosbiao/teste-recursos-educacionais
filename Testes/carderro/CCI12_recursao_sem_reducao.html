CCI12 Construir subprogramas recursivos Erro de progressão

Chamada recursiva sem redução do problema

Este card ajuda a reconhecer, explicar e corrigir uma recursão que possui caso base, mas não avança em direção a ele.

Gravidade Alto

O erro pode impedir completamente a finalização da função.

Sintoma observado

A função até tem caso base, mas nunca se aproxima dele.

Impacto no programa

A chamada recursiva repete o mesmo problema, cria novas ativações indefinidamente e pode causar travamento ou estouro de pilha antes de produzir o resultado.

Enunciado analisado

Escreva uma função recursiva em C para calcular o fatorial de um número inteiro n. Considere que fatorial(0) é igual a 1.

Este enunciado será usado como referência diagnóstica: cada chamada recursiva deve aproximar o valor de n do caso base definido.

Causa provável

O aluno entende que a função precisa chamar a si mesma, mas não ajusta o parâmetro para representar uma versão menor do problema.

  • A chamada usa o mesmo valor de entrada.
  • O caso base existe, mas fica inalcançável para valores maiores.

Evidência no código

  • Padrão detectável: dentro de fatorial(int n), a chamada aparece como fatorial(n).
  • Comportamento observado: para n maior que 0, a função repete a mesma chamada e não chega a n == 0.
  • Teste revelador: executar com n igual a 3 e rastrear se a próxima chamada usa 2 ou continua usando 3.

Quando aparece

Quando o aluno chama a função com o mesmo valor de entrada, sem aproximar o parâmetro do caso base.

  • No fatorial, chama fatorial(n) em vez de fatorial(n - 1).
  • Na soma recursiva, chama soma(n) em vez de reduzir n.
Exemplo com erro Código com erro 
#include <stdio.h>

int fatorial(int n) {
    if (n == 0) {
        return 1;
    }

    return n * fatorial(n); // erro: n não muda
}

int main() {
    printf("%d", fatorial(3));
    return 0;
}

O caso base existe, mas a chamada recursiva usa novamente o mesmo valor de n. Se n começa em 3, a próxima chamada também recebe 3, então a função não progride até n ser 0.

Correção mínima Código corrigido 
#include <stdio.h>

int fatorial(int n) {
    if (n == 0) {
        return 1;
    }

    return n * fatorial(n - 1);
}

int main() {
    printf("%d", fatorial(3));
    return 0;
}

A correção altera apenas a chamada recursiva. Agora cada chamada recebe uma versão menor do problema: 3 chama 2, 2 chama 1, 1 chama 0 e o caso base pode ser alcançado.

Entrada, processamento e saída

Parte do problema O que o enunciado pede O que o código com erro fez Diagnóstico
Entrada Receber um valor n para calcular seu fatorial. Recebe n, mas reutiliza esse mesmo n em todas as chamadas. A entrada inicial não é transformada em subproblemas menores.
Processamento Reduzir o problema até o caso base, como fatorial(3), fatorial(2), fatorial(1), fatorial(0). Repete fatorial(3), fatorial(3), fatorial(3) e assim por diante. A recursão não possui progressão efetiva.
Saída Retornar o resultado final, por exemplo 6 para fatorial(3). Não conclui o cálculo para valores maiores que 0. O programa pode travar ou encerrar por excesso de chamadas.

Antes e depois

Lógica incorreta

fatorial(3) chama fatorial(3)

O problema se repete com o mesmo tamanho. A distância até o caso base não diminui.

Lógica correta

fatorial(3) chama fatorial(2)

O problema fica menor a cada chamada. A sequência se aproxima de fatorial(0).

Por que isso acontece

A chamada recursiva precisa resolver uma versão menor ou mais simples do problema. Sem essa redução, a função repete a mesma tarefa indefinidamente.

Como identificar

Veja se o parâmetro muda em direção ao caso base.

Teste mínimo: para n igual a 3, escreva a próxima chamada. Se ela também for 3, há erro de progressão.

Forma correta

Ajustar a chamada para reduzir progressivamente o problema.

  1. Identifique o caso base.
  2. Veja qual parâmetro controla a parada.
  3. Garanta que a próxima chamada se aproxime desse caso.
  4. Teste a sequência de chamadas antes de executar.

Perguntas de verificação

Por que uma chamada recursiva precisa reduzir o problema?

Porque a redução aproxima a função do caso base. Sem isso, a recursão repete a mesma situação e pode nunca terminar.

Qual sinal no código indica esse erro no fatorial?

Um sinal claro é encontrar fatorial(n) dentro da própria função, quando o caso base depende de n diminuir até 0.

Se a função recebe n igual a 5, qual deveria ser a próxima chamada no fatorial?

A próxima chamada deve ser fatorial(4), pois n precisa se aproximar do caso base.

Teste rápido

Mini enunciado: implemente uma função recursiva que calcule a soma de 1 até n. Para n igual a 1, a função deve retornar 1.

int soma(int n) {
    if (n == 1) {
        return 1;
    }

    return n + soma(n); // erro: a próxima chamada não reduz n
}
