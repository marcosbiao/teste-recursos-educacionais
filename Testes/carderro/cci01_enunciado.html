CCI01 Representar problemas por soluções algorítmicas claras Erro de interpretação

Solução começa sem entender o problema

Este card ajuda a reconhecer quando a solução foi construída antes da leitura adequada do enunciado e mostra como corrigir o erro pela análise de entrada, processamento e saída.

Gravidade

Alto

Erro que pode comprometer toda a solução algorítmica.

Sintoma observado

O algoritmo resolve outra coisa

Mesmo funcionando parcialmente, a solução não responde exatamente ao problema proposto.

Impacto no programa

Resultado incoerente com o enunciado

O programa pode ler dados insuficientes, aplicar uma regra incompleta ou produzir uma saída que parece correta, mas não corresponde ao que foi pedido.

Enunciado analisado

Problema de referência

Calcule a média de três notas e mostre o resultado final.

Use este enunciado como referência para comparar o que foi pedido com o que o código realmente faz.

Causa provável

Pressa para começar o algoritmo

Esse erro costuma aparecer quando o aluno transforma uma lembrança parcial do problema em código, sem verificar antes quais dados entram, qual transformação deve ocorrer e qual resultado precisa ser entregue.

Evidência no código

Sinais observáveis

  • Padrão detectávelO enunciado pede 3 notas, mas o código declara e lê apenas nota1 e nota2.
  • Comportamento observadoA execução gera uma média, mas ela representa apenas duas notas.
  • Teste reveladorInformar três notas e verificar se todas participam do cálculo final.

Quando aparece

Antes da definição do problema

Ocorre quando o aluno começa a escrever passos sem identificar entrada, processamento e saída. Em exercícios de média, por exemplo, ele pode considerar apenas parte das notas necessárias.

Código com erro

Exemplo com erro

#include <stdio.h>

int main() {
    float nota1, nota2, media;

    scanf("%f", &nota1);
    scanf("%f", &nota2);

    // Problema: o enunciado pede 3 notas, mas apenas 2 foram lidas.
    media = (nota1 + nota2) / 2;

    printf("Media: %.2f", media);

    return 0;
}

O enunciado pede a média de três notas, mas o código lê apenas duas. A solução funciona parcialmente, porém resolve uma tarefa diferente da solicitada.

O erro não está apenas na fórmula. Ele começa na interpretação incompleta do problema.

Código corrigido

Correção mínima

#include <stdio.h>

int main() {
    float nota1, nota2, nota3, media;

    scanf("%f", &nota1);
    scanf("%f", &nota2);
    scanf("%f", &nota3);

    media = (nota1 + nota2 + nota3) / 3;

    printf("Media: %.2f", media);

    return 0;
}

A correção inclui a terceira nota porque ela faz parte da entrada exigida pelo problema. O processamento também passa a usar as três notas e divide a soma por três.

Essa mudança alinha entrada, processamento e saída ao que foi pedido no enunciado.

Comparação

Antes e depois

A diferença essencial está em verificar o problema antes de escrever a solução.

Lógica incorreta

ler nota1
ler nota2
media = (nota1 + nota2) / 2

A solução assume que o problema tem duas notas, mas não confirma essa regra no enunciado.

Lógica correta

ler nota1
ler nota2
ler nota3
media = (nota1 + nota2 + nota3) / 3

A solução considera todas as entradas exigidas e aplica o processamento correspondente.

Por que isso acontece

Entendimento antes do código

Antes de programar, é preciso compreender o problema, identificar os dados disponíveis, a transformação esperada e o resultado final. Sem essa leitura inicial, o aluno pode escrever um algoritmo organizado para o problema errado.

Como identificar

Estratégia diagnóstica prática

Verifique se a solução responde exatamente ao que foi pedido. Uma forma simples é comparar o enunciado com a lista de entradas, regras e saídas presentes no código.

Pergunta guia O que observar
Quais dados entram? O código lê todos os valores necessários?
Qual regra processa os dados? A fórmula considera todas as informações exigidas?
Qual saída deve aparecer? O resultado exibido corresponde ao enunciado?

Forma correta

Procedimento de correção

Reler o enunciado, destacar entradas, saídas e regras antes de montar o algoritmo. Depois disso, cada linha do código deve ter relação direta com uma parte do problema.

  1. Leia o enunciado inteiro.
  2. Marque os dados de entrada.
  3. Marque a regra de processamento.
  4. Marque a saída esperada.
  5. Escreva o algoritmo somente depois dessa análise.

Verificação

Perguntas de verificação

Por que começar pelo código pode causar esse erro?

Porque o aluno pode escrever comandos baseados em uma ideia incompleta do problema. Antes do código, é necessário identificar entradas, processamento e saída.

Como perceber que o algoritmo está resolvendo outra coisa?

Compare o enunciado com o comportamento do programa. Se o código ignora dados necessários ou usa uma regra diferente da pedida, ele está resolvendo outro problema.

O que deve ser feito antes de montar o algoritmo?

Reler o enunciado, destacar os dados de entrada, identificar as regras de processamento e definir a saída esperada.
