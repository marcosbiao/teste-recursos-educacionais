Ideia central

Arquivo exige um ciclo seguro

Em C, um arquivo não deve ser usado diretamente. O programa precisa criar uma referência com FILE *, abrir o arquivo com fopen, verificar se a abertura funcionou, realizar a leitura ou escrita e finalizar com fclose.

1

Abrir com intenção

Escolha o arquivo e o modo adequado: leitura, escrita ou acréscimo.

2

Verificar antes de usar

Confirme se fopen não retornou NULL.

3

Fechar ao terminar

Use fclose para liberar o recurso e concluir o acesso.

Como pensar o ciclo do arquivo

Etapa 1

Declarar a referência

Use FILE * para guardar a referência ao arquivo que será acessado.

Etapa 2

Abrir com fopen

Informe o nome do arquivo e o modo de abertura, como "r", "w" ou "a".

Etapa 3

Verificar NULL

Se a abertura falhar, o ponteiro recebe NULL. O programa deve tratar esse caso.

Etapa 4

Ler, escrever e fechar

Use funções como fscanf ou fprintf e finalize com fclose.

"r"

Ler arquivo existente

Use quando o programa precisa consultar dados já gravados.

"w"

Criar ou sobrescrever

Use quando o programa deve gravar um novo conteúdo no arquivo.

"a"

Acrescentar ao final

Use quando o programa deve manter o conteúdo anterior e adicionar novos dados.

Ponto de atenção: abrir um arquivo com "w" pode apagar o conteúdo anterior. Antes de usar esse modo, verifique se a intenção é realmente substituir o arquivo.

Exemplo mínimo

Problema

Gravar a média de um estudante no arquivo dados.txt.

Parte Aplicação no exemplo
Referência FILE *arq guarda o acesso ao arquivo.
Abertura fopen("dados.txt", "w") abre para escrita.
Verificação arq == NULL indica falha na abertura.
Escrita e fechamento fprintf grava o texto e fclose encerra o acesso.

Código curto em C

#include <stdio.h>

int main() {
    float media = 8.5;
    FILE *arq = fopen("dados.txt", "w");

    if (arq == NULL) {
        printf("Erro ao abrir arquivo");
        return 1;
    }

    fprintf(arq, "Media: %.2f\n", media);
    fclose(arq);

    return 0;
}

Erros comuns

Usar o arquivo sem verificar

Por que acontece: o aluno chama fprintf ou fscanf logo após o fopen.

Como evitar: teste se o ponteiro é NULL antes de qualquer leitura ou escrita.

Trocar o modo de abertura

Por que acontece: o aluno usa "w" quando precisava apenas ler ou acrescentar dados.

Como evitar: defina a intenção antes de abrir: ler, substituir ou acrescentar.

Esquecer o fclose

Por que acontece: o aluno entende a gravação como última etapa e esquece de finalizar o acesso.

Como evitar: associe todo fopen bem sucedido a um fclose ao final.

Perguntas rápidas

Por que o programa deve verificar se fopen retornou NULL?

Porque NULL indica que o arquivo não foi aberto corretamente. Usar o arquivo nessa situação pode causar erro no programa.

Quando o modo "a" é mais adequado que "w"?

Quando o programa precisa acrescentar novas informações ao final do arquivo sem apagar o conteúdo anterior.

Qual é a função de fclose no ciclo de arquivo?

fclose encerra o acesso ao arquivo e libera o recurso usado pelo programa.
