O ciclo básico de um arquivo em C

Abrir, verificar, ler ou escrever, e fechar o arquivo de forma segura.

Ideia central

Um arquivo permite que os dados continuem existindo depois que o programa termina. Para isso, o programa precisa seguir um ciclo seguro de acesso ao arquivo.

1

Abrir

fopen()

Escolha o arquivo e o modo de abertura.

2

Verificar

arquivo != NULL

Confirme se o arquivo foi aberto antes de usar.

3

Ler ou escrever

fscanf() ou fprintf()

Leia dados existentes ou grave novos dados.

4

Fechar

fclose()

Libere o recurso e finalize o acesso.

Modos de abertura
"r" Ler arquivo existente.
"w" Criar ou sobrescrever.
"a" Acrescentar ao final.

Atenção: abrir com "w" pode apagar o conteúdo anterior do arquivo.

Exemplo mínimo
FILE *arq = fopen("dados.txt", "w");

if (arq == NULL) {
    printf("Erro ao abrir arquivo");
    return 1;
}

fprintf(arq, "Media: %.2f\n", media);
fclose(arq);
Checklist de segurança
O ponteiro FILE * foi declarado?
O modo de abertura combina com a tarefa?
O programa verificou se fopen retornou NULL?
A leitura ou escrita usa o formato correto?
O arquivo é fechado com fclose no final?
O resultado pode ser conferido no arquivo gerado?
Erros comuns:

Usar o arquivo sem verificar se abriu, esquecer o fclose, trocar "r" por "w", ou gravar dados sem um formato claro para depois conseguir ler.
