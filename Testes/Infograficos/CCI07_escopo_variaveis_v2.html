Ideia central

Variável local pertence ao bloco onde foi criada

Uma variável declarada dentro de uma função, procedimento ou bloco só pode ser usada naquele espaço de código. Esse limite é chamado de escopo e ajuda a evitar confusão entre partes diferentes do programa.

1

Criação

A variável nasce quando é declarada dentro de um bloco.

2

Uso

Ela pode ser lida e alterada apenas dentro do seu escopo.

3

Limite

Fora do bloco, o nome daquela variável deixa de existir para o programa.

As chaves delimitam o espaço

O escopo acompanha o bloco onde a variável foi declarada. Fora dele, aquele nome não está disponível.

→

Parâmetros levam valores

Quando main chama uma função, os valores podem ser enviados por parâmetros. A função usa seus próprios nomes locais.

↩

Retorno traz resultado

A função não precisa expor sua variável local. Ela calcula internamente e devolve apenas o valor final.

Aplicando o conceito

Passo 1

Localize a declaração

Veja em qual função ou bloco a variável foi criada.

Passo 2

Defina o limite

O escopo acompanha as chaves { } onde a variável foi declarada.

Passo 3

Passe dados corretamente

Use parâmetros para levar valores até uma função e retorno para trazer resultado.

Passo 4

Evite global sem motivo

Prefira variáveis locais quando o dado só faz sentido em uma parte do programa.

Exemplo mínimo

Problema

Calcular a média de duas notas usando uma função. A leitura fica em main e o cálculo fica na função.

Parte Aplicação no exemplo
Variáveis locais de main n1, n2 e m existem apenas dentro de main.
Parâmetros da função a e b recebem os valores enviados por main.
Variável local da função resultado existe apenas dentro da função media.
Retorno A função devolve o valor calculado para main.

Código curto em C

variável local fluxo de valor 
#include <stdio.h>

float media(float a, float b) {
    float resultado;
    resultado = (a + b) / 2;
    return resultado;
}

int main() {
    float n1, n2, m;

    scanf("%f", &n1);
    scanf("%f", &n2);

    m = media(n1, n2);

    printf("Media: %.2f", m);

    return 0;
}

Erros comuns

Usar variável local fora do escopo

Por que acontece: o aluno declara uma variável dentro da função e tenta acessá-la em outra parte do programa.

Como evitar: verifique em qual bloco a variável foi criada e use retorno ou parâmetro quando precisar comunicar dados.

Criar global para resolver acesso

Por que acontece: a variável global parece uma solução rápida para ser vista por todas as funções.

Como evitar: prefira variáveis locais e passagem de valores por parâmetros.

Confundir parâmetro com variável original

Por que acontece: o aluno acha que o nome usado na função precisa ser igual ao nome usado em main.

Como evitar: entenda que parâmetros recebem valores enviados na chamada, mesmo com nomes diferentes.

Perguntas rápidas

Uma variável declarada dentro de uma função pode ser usada diretamente em outra função?

Não. Ela é local da função onde foi declarada. Para compartilhar um valor, use parâmetro, retorno ou outra estratégia justificada.

No exemplo, a variável resultado existe dentro de main?

Não. A variável resultado foi declarada dentro da função media. O que chega em main é o valor retornado pela função.

Quando uma variável global pode ser inadequada para um iniciante?

Quando ela é usada apenas para evitar passagem de parâmetros. Isso pode esconder dependências e dificultar a compreensão do fluxo do programa.
