Ideia central

Um programa introdutório é uma transformação

O aluno deve observar o caminho dos dados: primeiro eles entram no programa, depois são processados e, por fim, geram uma saída compreensível.

E Entrada ler ou receber dados
→
P Processamento calcular, comparar, transformar
→
S Saída mostrar o resultado

Regra prática

Antes de escrever código, tente responder três perguntas: quais dados entram, o que será feito com eles e o que precisa aparecer ao final.

1. O que o programa recebe? 2. O que o programa calcula ou decide? 3. O que o programa mostra?

C

Exemplo mínimo em C

Problema

Ler duas notas, calcular a média e mostrar o resultado.

Entrada: nota1 e nota2 Processamento: (nota1 + nota2) / 2 Saída: média exibida no console

Código curto

float nota1, nota2, media;

scanf("%f", &nota1);
scanf("%f", &nota2);

media = (nota1 + nota2) / 2;

printf("Media: %.2f", media);

Leitura pelo fluxo E P S

Os comandos scanf representam a entrada. A atribuição de media representa o processamento. O printf representa a saída. Essa ordem torna o programa verificável.

!

Erros comuns

Mostrar antes de calcular

Por que acontece: o aluno tenta exibir uma variável antes de ela receber o resultado.

Como evitar: confirme se o processamento ocorreu antes do printf.

Ler dados desnecessários

Por que acontece: o aluno não identifica exatamente o que o enunciado pede.

Como evitar: liste apenas os dados necessários para resolver o problema.

Confundir entrada com saída

Por que acontece: o aluno trata o resultado final como se fosse um dado inicial.

Como evitar: pergunte se o valor é fornecido ao programa ou produzido por ele.

Processamento incompleto

Por que acontece: o aluno lê os dados, mas não realiza a transformação necessária.

Como evitar: verifique qual cálculo, comparação ou decisão liga entrada e saída.
