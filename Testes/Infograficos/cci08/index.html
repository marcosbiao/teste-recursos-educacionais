Ideia central

Justificar é ligar três pontos

Uma justificativa técnica mostra que a escolha feita no código responde a uma exigência do problema.

1 Enunciado O que foi pedido?
2 Código O que foi escolhido?
3 Evidência Por que a escolha faz sentido?
Resposta fraca

“Usei if porque funcionou.”

Resposta técnica

“Usei if porque o enunciado pede uma decisão: mostrar uma mensagem diferente conforme a nota.”

C

Exemplo mínimo em C

Enunciado

Ler duas notas e informar se o aluno foi aprovado. Considere aprovado quem tem média maior ou igual a 7.

Dado relevante: duas notas Critério: média ≥ 7 Saída: aprovado ou não aprovado

Código

media = (nota1 + nota2) / 2;

if (media >= 7) {
    printf("Aprovado");
} else {
    printf("Nao aprovado");
}

Justificativa técnica

A média é calculada somando as duas notas e dividindo por 2 porque o enunciado informa exatamente duas notas. A condição media >= 7 foi usada porque o critério de aprovação definido no problema é média maior ou igual a 7. O else cobre os demais casos, nos quais a média fica abaixo do limite de aprovação.

!

Erros comuns

Dizer apenas que funcionou

Por que acontece: o aluno confunde resultado correto com justificativa técnica.

Como evitar: explique qual parte do enunciado é atendida pela escolha feita.

Justificar sem citar o código

Por que acontece: a explicação fica genérica e não mostra evidência.

Como evitar: mencione a variável, condição, fórmula ou estrutura usada.

Repetir o código em palavras

Por que acontece: o aluno descreve o que o código faz, mas não explica por que ele faz sentido.

Como evitar: relacione a escolha com o pedido do problema.

Usar argumento de autoridade

Por que acontece: aparecem frases como “porque o professor ensinou assim”.

Como evitar: baseie a justificativa em evidências do enunciado e do código.
