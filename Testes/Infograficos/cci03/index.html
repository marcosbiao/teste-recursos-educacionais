Ideia central

Variável é um espaço nomeado na memória

Uma variável permite que o programa guarde um valor e use esse valor depois. O nome da variável funciona como uma identificação para acessar o conteúdo armazenado.

nome idade
endereço simbólico 18 valor armazenado
→

Constante é valor protegido

Uma constante também tem nome e valor, mas seu papel é representar um dado que não deve ser alterado durante a execução do programa.

const float PI 3.14 valor fixo

Variável, constante e atribuição

V

Variável

Tem nome, tipo e valor armazenado. Pode receber novos valores ao longo do programa.

int idade = 18;
idade = 19;
C

Constante

Tem nome e valor fixo. Deve representar dados que não serão alterados.

const float PI = 3.14;
=

Atribuição

Atualiza o conteúdo de uma variável. O lado direito é calculado antes de guardar o resultado.

total = total + 1;
C

Exemplo mínimo em C

Problema

Guardar a idade de uma pessoa, atualizar essa idade e mostrar o novo valor.

Variável: idade Valor inicial: 18 Atribuição: idade = idade + 1 Novo estado: 19

Código curto

int idade = 18;
const int MAIORIDADE = 18;

idade = idade + 1;

printf("Idade: %d", idade);
printf("Maioridade: %d", MAIORIDADE);

Leitura técnica

A variável idade começa com valor 18 e depois recebe um novo valor calculado a partir do estado anterior. A constante MAIORIDADE representa um valor fixo usado como referência e não deve ser alterada.

!

Erros comuns

Tratar variável como número solto

Por que acontece: o aluno vê apenas o valor e esquece que ele está guardado em um nome.

Como evitar: sempre pergunte qual variável guarda esse valor.

Confundir atribuição com igualdade matemática

Por que acontece: o símbolo = parece uma igualdade comum.

Como evitar: leia como “recebe”: x = x + 1 significa x recebe o valor anterior de x mais 1.

Alterar uma constante

Por que acontece: o aluno não distingue valor fixo de valor variável.

Como evitar: use constante apenas para valores que devem permanecer estáveis.

Perder o estado atual

Por que acontece: o aluno não acompanha as mudanças após cada comando.

Como evitar: simule linha por linha qual valor fica armazenado depois de cada atribuição.
