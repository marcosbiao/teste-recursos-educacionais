Qual laço usar?

Compare for, while e do while para escolher uma repetição adequada ao problema antes de escrever o código em C.

Ideia central

O laço depende do controle da repetição

Antes de escolher um laço, observe se você sabe quantas vezes a repetição ocorrerá, se a condição deve ser testada antes da execução ou se o bloco precisa executar pelo menos uma vez.

1

Quantidade conhecida

Use for quando há contador claro e limite definido.

2

Condição antes

Use while quando a repetição depende de uma condição inicial.

3

Execução mínima

Use do while quando o bloco deve ocorrer ao menos uma vez.

Como escolher o laço

for

Repetição contada

Indicado quando existe início, condição e atualização bem definidos no próprio comando.

while

Teste antes do bloco

Indicado quando a repetição pode nem começar, pois a condição é verificada antes da primeira execução.

do while

Teste depois do bloco

Indicado quando é necessário executar uma ação inicial e só depois verificar se ela deve repetir.

Situação Laço mais adequado Motivo
Mostrar os números de 1 a 10 for A quantidade de repetições é conhecida e o contador é claro.
Ler valores enquanto forem positivos while A repetição depende de uma condição que precisa ser testada antes.
Exibir um menu e repetir enquanto o usuário desejar do while O menu precisa aparecer pelo menos uma vez antes da decisão de repetir.

Exemplo mínimo

Problema

Mostrar os números de 1 a 5. Como a quantidade de repetições é conhecida, o laço for é uma escolha adequada.

Parte Aplicação no exemplo
Controle Variável i atua como contador
Início i começa em 1
Parada Repetir enquanto i for menor ou igual a 5
Atualização i aumenta uma unidade por repetição

Código curto em C

#include <stdio.h>

int main() {
    int i;

    for (i = 1; i <= 5; i++) {
        printf("%d\n", i);
    }

    return 0;
}

Erros comuns

Usar for sem limite claro

Por que acontece: o aluno tenta usar contador mesmo quando a repetição depende de uma condição aberta.

Como evitar: verifique se a quantidade de repetições é conhecida antes de escolher for.

Esquecer a atualização

Por que acontece: a variável de controle não muda dentro do laço ou na expressão de atualização.

Como evitar: confira se algum comando aproxima a repetição da condição de parada.

Escolher do while sem necessidade

Por que acontece: o aluno não percebe que o bloco será executado pelo menos uma vez.

Como evitar: use do while apenas quando a primeira execução deve ocorrer antes do teste.

Perguntas rápidas

Qual laço costuma ser mais adequado quando a quantidade de repetições é conhecida?

O laço for, porque ele organiza início, condição e atualização em uma estrutura compacta.

Quando um while pode executar zero vezes?

Quando a condição já é falsa antes da primeira execução do bloco.

Por que o do while é comum em menus simples?

Porque o menu precisa aparecer pelo menos uma vez antes de verificar se o usuário deseja continuar.
