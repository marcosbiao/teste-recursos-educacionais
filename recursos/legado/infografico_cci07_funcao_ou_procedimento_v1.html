Ideia central

Modularizar é separar responsabilidades

Uma função com retorno calcula ou produz um valor que será usado depois. Um procedimento void executa uma ação e não devolve valor para quem chamou.

1

Há valor a devolver?

Use função com retorno quando o resultado precisa ser guardado ou usado em outra expressão.

2

É apenas uma ação?

Use procedimento void quando a tarefa é mostrar, organizar ou executar algo sem devolver valor.

3

A chamada deve fazer sentido

Observe se a chamada será usada como valor ou apenas como uma instrução no programa.

Como escolher entre função e procedimento

Pergunta 1

O módulo calcula algo?

Se o módulo produz um resultado, prefira uma função com retorno.

Pergunta 2

O resultado será usado?

Se o valor entra em outra conta, comparação ou atribuição, ele precisa ser retornado.

Pergunta 3

O módulo só executa?

Se a tarefa é apenas exibir uma mensagem ou imprimir um resultado, void pode ser suficiente.

Pergunta 4

A responsabilidade está clara?

Evite colocar cálculo, leitura e exibição no mesmo módulo quando isso prejudicar a clareza.

Tipo de módulo Quando usar Como aparece na chamada
Função com retorno Quando há um valor a devolver resultado = dobro(x);
Procedimento void Quando a tarefa apenas executa uma ação mostrarResultado(resultado);

Exemplo mínimo

Problema

Calcular o dobro de um número e mostrar o resultado.

Parte Aplicação no exemplo
Função com retorno dobro calcula e devolve um valor inteiro.
Procedimento void mostrarResultado apenas exibe o valor recebido.
Programa principal main chama os módulos na ordem necessária.

Código curto em C

#include <stdio.h>

int dobro(int valor) {
    return valor * 2;
}

void mostrarResultado(int numero) {
    printf("Resultado: %d\n", numero);
}

int main() {
    int x = 4;
    int resultado;

    resultado = dobro(x);
    mostrarResultado(resultado);

    return 0;
}

Erros comuns

Esperar retorno de void

Por que acontece: o aluno usa um procedimento como se ele devolvesse um valor.

Como evitar: use void apenas quando a tarefa não precisa retornar resultado.

Esquecer o return

Por que acontece: a função foi declarada com tipo de retorno, mas não devolve o valor calculado.

Como evitar: toda função com retorno deve ter um return coerente com o tipo declarado.

Fazer tudo no main

Por que acontece: o aluno não separa responsabilidades e concentra todas as ações em um único bloco.

Como evitar: crie módulos pequenos para cálculos, exibições ou tarefas repetidas.

Perguntas rápidas

Quando devo usar uma função com retorno?

Quando o módulo precisa produzir um valor que será guardado, comparado, impresso ou usado em outra expressão.

O que significa usar void em um procedimento?

Significa que o procedimento executa uma ação, mas não devolve um valor para quem fez a chamada.

No exemplo, por que dobro não é void?

Porque ele calcula um valor e esse valor precisa voltar para o programa principal para ser armazenado em resultado.
