Variável não inicializada

Investigue como um cálculo aparentemente simples se torna imprevisível quando uma variável é usada antes de receber um valor inicial coerente com seu papel.

Etapa 1 · Observe

Localize o primeiro uso da variável e verifique qual estado ela possui

Enunciado

Leia cinco números inteiros, calcule a soma de todos eles e apresente o total.

Onde está o erro?

Análise do diagnóstico

Qual é o estado de soma antes da primeira atualização?

Diagnóstico

A alternativa A descreve o erro: declarar uma variável reserva espaço para ela, mas não define automaticamente um valor inicial confiável.

Causa conceitual

Na expressão soma = soma + numero, o valor anterior de soma precisa ser lido. Como ele nunca foi definido, o estado usado pelo cálculo é indeterminado e o comportamento do programa não é confiável.

Quando aparece

É comum em acumuladores, contadores, auxiliares, flags e variáveis de máximo ou mínimo quando o estudante começa a atualizá-las sem estabelecer o estado inicial.

Como o erro se propaga

O primeiro estado desconhecido afeta todas as atualizações posteriores.

Como corrigir?

Etapa 3 · Corrija

Defina um estado inicial coerente antes do primeiro uso

Correção mínima em C
−int numero, soma;
+int numero, soma = 0;
Por que funciona?

Como soma é um acumulador aditivo, zero é o elemento neutro da soma. A primeira atualização passa a combinar o número lido com um estado conhecido.

  • Identifique quais variáveis serão lidas antes de receber outra atribuição.
  • Inicialize acumuladores de soma com zero.
  • Inicialize contadores com zero.
  • Escolha o valor inicial de acordo com o papel da variável.
  • Para máximo e mínimo, considere usar o primeiro dado válido, não um valor arbitrário.
Valide a aprendizagem

Etapa 4 · Valide

Reconheça, aplique e analise o uso de variáveis sem estado inicial

Passe pelas abas e observe qual variável é consultada antes de receber um valor confiável. Nem toda variável deve começar em zero.

Exemplos concluídos: 0 de 3.
Reconheça

Contagem de números positivos

Leia quatro números inteiros e mostre quantos deles são positivos.

#include <stdio.h> int main() { int numero, positivos; for (int i = 0; i < 4; i++) { scanf("%d", &numero); if (numero > 0) { positivos++; } } printf("Positivos: %d\n", positivos); return 0; }

Maior valor com números negativos

Leia cinco números inteiros, que podem ser negativos, e apresente o maior valor informado.

#include <stdio.h> int main() { int numero, maior; for (int i = 0; i < 5; i++) { scanf("%d", &numero); if (numero > maior) { maior = numero; } } printf("Maior: %d\n", maior); return 0; }

Qual é o diagnóstico?

Síntese diagnóstica

Uma variável só participa de um cálculo de forma confiável depois que seu estado inicial foi definido de acordo com o papel que ela exerce.

Erro

Usar uma variável antes de atribuir um valor inicial.

Sinal

Resultados mudam, parecem aleatórios ou dependem do ambiente.

Diagnóstico

Rastrear a primeira leitura e a primeira atribuição de cada variável.

Correção

Inicializar cada estado com um valor coerente com sua função.
