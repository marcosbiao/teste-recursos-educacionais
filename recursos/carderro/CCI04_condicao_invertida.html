Etapa 1 · Observe

Compare a regra de maioridade com a condição implementada

Enunciado

Onde está o erro?

Análise do diagnóstico

A condição representa a mesma regra descrita no enunciado?

Diagnóstico

A alternativa A descreve o erro. O bloco “Maior de idade” deveria executar quando idade >= 18, mas o código usa idade < 18.

Causa conceitual

Uma condição não deve apenas compilar: ela precisa representar exatamente a regra de decisão. Trocar maior por menor ou incluir o limite no lado errado inverte o conjunto de casos selecionados.

Como identificar

Reescreva a condição em linguagem natural e teste um valor abaixo do limite, o próprio limite e um valor acima dele.

Como o erro se propaga

A condição verdadeira conduz ao bloco semanticamente oposto.

Etapa 3 · Corrija

Traduza a regra para linguagem natural antes de escolher os operadores

Correção mínima em C
−if (idade < 18) {
+if (idade >= 18) {
+ printf("Maior de idade ");
+} else {
+ printf("Menor de idade ");
+}
Por que funciona?

“Maior de idade” corresponde à frase “idade maior ou igual a 18”. O operador >= inclui corretamente o caso-limite 18 no primeiro bloco.

  • Escreva em palavras quando o primeiro bloco deve executar.
  • Identifique se o limite deve ser incluído.
  • Converta cada relação para o operador correspondente.
  • Teste valores abaixo, no limite e acima dele.
  • Confirme se a mensagem do bloco combina com a condição.
