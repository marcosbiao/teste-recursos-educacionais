Etapa 1 · Observe

O programa tenta ler antes de saber se existe um arquivo válido

Problema

Abra o arquivo alunos.txt, leia o primeiro nome armazenado e apresente esse nome na tela. Caso o arquivo não possa ser aberto, o programa deve informar o problema sem tentar realizar a leitura.

Solução em análise

Se fopen falhar, arquivo receberá NULL. Mesmo assim, o programa entrega esse valor a fscanf, tentando operar sobre um arquivo que não foi aberto.

Onde está o erro?

Análise do diagnóstico

A abertura do arquivo é uma tentativa que pode falhar

Diagnóstico

A alternativa A descreve o erro: o programa não confirma se fopen devolveu um ponteiro válido antes de executar fscanf.

Causa conceitual

Abrir um arquivo depende do sistema operacional, do caminho, das permissões e do modo solicitado. Por isso, a abertura não é garantida.

Quando aparece

Quando o código pressupõe que o arquivo sempre existe e está acessível, tratando a chamada de fopen como se não pudesse falhar.

Fluxo seguro de uso do arquivo

A verificação deve acontecer antes da primeira leitura ou gravação.

Como corrigir?

Etapa 3 · Corrija

Interrompa o fluxo quando a abertura não produzir um arquivo válido

Correção mínima em C
−FILE *arquivo = fopen("alunos.txt", "r");
−fscanf(arquivo, "%49s", nome);
+FILE *arquivo = fopen("alunos.txt", "r");
+if (arquivo == NULL) {
+ printf("Erro ao abrir o arquivo.\n");
+ return 1;
+}
+fscanf(arquivo, "%49s", nome);
Por que funciona?

A verificação cria dois caminhos explícitos: continuar somente com um arquivo válido ou encerrar com uma mensagem quando a abertura falhar.

  • Armazenar o retorno de fopen em um ponteiro FILE *.
  • Comparar o ponteiro com NULL imediatamente após a abertura.
  • Não executar leitura ou escrita quando a abertura falhar.
  • Informar o erro ou devolver um código que permita tratá-lo.
  • Fechar somente os arquivos que foram abertos com sucesso.

Qual análise está correta?

Síntese diagnóstica

Operações de persistência só devem começar depois que a abertura do recurso for confirmada.

Erro

Usar o ponteiro do arquivo sem verificar o retorno de fopen.

Sinal

O programa falha quando o arquivo não existe ou está inacessível.

Diagnóstico

Procurar uma comparação com NULL antes da primeira operação.

Correção

Tratar a falha de abertura antes de ler, gravar ou fechar o arquivo.
