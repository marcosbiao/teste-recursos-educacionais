Cadastro sem validação mínima

Investigue por que ler e armazenar dados imediatamente pode inserir nomes vazios, idades negativas e outros registros incoerentes no sistema.

SintomaDados incoerentes são aceitos

Etapa 1 · Observe

O programa registra os dados antes de verificar se eles são aceitáveis

Problema

Cadastre o nome e a idade de uma pessoa. O nome é obrigatório e a idade não pode ser negativa. Dados inválidos não devem ser armazenados.

Solução em análise
Linguagem C
1#include <stdio.h>
2#include <string.h>
3
4typedef struct {
5 char nome[50];
6 int idade;
7} Pessoa;
8
9int main() {
10 Pessoa pessoa;
11
12 fgets(pessoa.nome, sizeof(pessoa.nome), stdin);
13 pessoa.nome[strcspn(pessoa.nome, "\n")] = '\0';
14 scanf("%d", &pessoa.idade);
15
16 // nenhum campo é validado antes do cadastro
17 printf("Cadastro salvo: %s, %d anos\n",
18 pessoa.nome, pessoa.idade);
19
20 return 0;
21}
O programa trata leitura como aceitação. Assim, qualquer conteúdo digitado segue diretamente para o cadastro, mesmo quando viola as regras do problema.

Onde está o erro?

Análise do diagnóstico

Dados digitados só se tornam dados do cadastro depois de serem validados

Diagnóstico

A alternativa A descreve o erro: o programa lê os campos e avança diretamente para o armazenamento, sem verificar as regras mínimas de aceitação.

Causa conceitual

Entrada e cadastro não são a mesma etapa. A leitura captura o que foi digitado; a validação decide se esse dado pode integrar o estado do sistema.

Quando aparece

Quando o estudante se preocupa apenas em preencher a estrutura e esquece que campos obrigatórios, faixas numéricas e formatos precisam ser verificados antes da gravação.

Fluxo seguro de um cadastro

A validação funciona como uma barreira entre a entrada bruta e os dados persistidos.

Etapa 3 · Corrija

Verifique todas as regras mínimas antes de confirmar o cadastro

Validação antes do armazenamento
−printf("Cadastro salvo: %s, %d anos\n", nome, idade);
+if (strlen(nome) == 0 || idade < 0) {
+ printf("Dados invalidos. Cadastro nao realizado.\n");
+ return 1;
+}
+printf("Cadastro salvo: %s, %d anos\n", nome, idade);
Por que funciona?

O armazenamento só é alcançado quando todas as regras mínimas são verdadeiras. Caso contrário, o programa informa o problema e mantém o cadastro inalterado.

NomeDeve conter pelo menos um caractere.
IdadeDeve ser maior ou igual a zero.
ArmazenamentoSó deve ocorrer depois que todas as regras forem verificadas.
  • Listar os campos obrigatórios do cadastro.
  • Definir os valores ou formatos aceitáveis.
  • Validar antes de alterar a coleção de registros.
  • Informar claramente qual campo está inválido.
  • Testar dados válidos, ausentes e fora da faixa.

Registro parcialmente validado

Qual análise está correta?

Síntese diagnóstica

Um cadastro confiável aceita apenas registros que atendem às regras mínimas definidas para cada campo.

Erro

Armazenar os dados imediatamente após a leitura.

Sinal

Nome vazio, número inválido ou formato incoerente entra no cadastro.

Diagnóstico

Procurar regras de aceitação antes da gravação.

Correção

Validar todos os campos obrigatórios e só então armazenar.
