Etapa 1 · Observe

O programa grava a nova venda, mas elimina todo o histórico anterior

Problema

A cada execução, leia o valor de uma venda e acrescente esse valor ao final do arquivo vendas.txt, preservando todos os registros já armazenados.

Solução em análise
Linguagem C
1#include <stdio.h>
2
3int main() {
4 FILE *arquivo = fopen("vendas.txt", "w");
5 float valor;
6
7 if (arquivo == NULL) {
8 printf("Erro ao abrir o arquivo.\n");
9 return 1;
10 }
11
12 scanf("%f", &valor);
13 fprintf(arquivo, "%.2f\n", valor);
14
15 fclose(arquivo);
16 return 0;
17}
O modo "w" cria o arquivo quando necessário, mas também reduz um arquivo existente a tamanho zero no momento da abertura. A nova venda é gravada depois que o histórico já foi apagado.

Onde está o erro?

Análise do diagnóstico

O modo de abertura determina o que acontece antes da primeira operação

Diagnóstico

A alternativa A descreve o erro. O modo "w" é adequado para criar ou substituir um arquivo, mas não para preservar um histórico.

Causa conceitual

O modo não informa apenas se haverá leitura ou escrita. Ele também define se o arquivo precisa existir, se será criado e se o conteúdo anterior será mantido ou truncado.

Quando aparece

Quando o estudante escolhe um modo por associá-lo apenas à palavra “escrita”, sem comparar seus efeitos com a intenção de ler, substituir ou acrescentar.

Como o histórico é perdido

A perda ocorre na abertura, antes de fprintf executar.

Etapa 3 · Corrija

Escolha o modo que preserva o conteúdo e posiciona a gravação no final

Correção mínima em C
−FILE *arquivo = fopen("vendas.txt", "w");
+FILE *arquivo = fopen("vendas.txt", "a");
+if (arquivo == NULL) {
+ printf("Erro ao abrir o arquivo. ");
+ return 1;
+}
+fprintf(arquivo, "%.2f ", valor);
Por que funciona?

O modo "a" cria o arquivo quando ele ainda não existe e direciona cada gravação para o final, sem eliminar o conteúdo armazenado anteriormente.

  • Definir se a tarefa exige leitura, escrita ou ambas.
  • Decidir se um arquivo existente deve ser preservado ou substituído.
  • Usar "r" para somente ler um arquivo existente.
  • Usar "w" apenas quando substituir o conteúdo for intencional.
  • Usar "a" quando novos dados devem ser acrescentados ao final.
  • Considerar os modos com + quando o mesmo fluxo precisa ler e escrever.

Síntese diagnóstica

Escolher um modo de abertura significa definir permissões e também o destino do conteúdo já armazenado.

Erro

Usar um modo incompatível com a operação pretendida.

Sinal

Dados desaparecem, não são lidos ou não podem ser gravados.

Diagnóstico

Comparar a intenção com os efeitos de r, w, a e seus modos de atualização.

Correção

Selecionar o modo que preserva, substitui ou acrescenta exatamente como o problema exige.
