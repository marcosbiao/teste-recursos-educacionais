Função ou procedimento? Entenda a diferença

Compare estruturas que devolvem um valor com estruturas void que executam uma ação, e escolha a forma adequada para cada parte do programa.

Mesma ideia: tarefa separadaA diferença aparece na forma de comunicar o resultado.
Função com retorno int dobro(int n)

Calcula e devolve um valor para ser usado em outra parte do programa.

↔
Procedimento void mostrar(int n)

Executa uma ação, mas não fornece um valor por meio de return.

Em C: os dois são funções. Neste infográfico, chamamos de procedimento uma função cujo tipo de retorno é void.

Ideia central

Separe tarefas e defina como cada módulo entrega seu resultado

Uma função ou procedimento reúne comandos relacionados a uma responsabilidade. A escolha depende de uma pergunta: o restante do programa precisa receber um valor calculado?

Quando a resposta for sim, use um tipo de retorno como int, float ou char. Quando a tarefa apenas executar uma ação, use void.
1

Receber

Parâmetros permitem que o módulo trabalhe com dados enviados na chamada.

2

Executar

O corpo concentra uma tarefa clara, sem misturar responsabilidades diferentes.

3

Comunicar

A função devolve um valor. O procedimento conclui uma ação sem valor de retorno.

Uma tarefa separada nome(dados) recebe uma chamada, executa comandos e comunica o resultado de uma forma definida.

Função com retorno

Produz um valor que pode entrar em uma atribuição, expressão ou condição.

return resultado;

Procedimento void

Executa uma ação, como mostrar uma mensagem, sem devolver valor à chamada.

void mostrar(...)

Parâmetros e argumentos

Os parâmetros recebem os dados. Os argumentos são os valores enviados na chamada.

dobro(numero)

Escolha pelo uso

Observe o que deve acontecer depois da chamada, não apenas o nome da tarefa.

valor = função(...)

Duas rotas para modularizar

O resultado será usado depois?

Escolha uma função com retorno.

media = calcularMedia(n1, n2);
  • o valor entra em uma variável;
  • pode participar de uma expressão;
  • pode ser testado em uma condição.
OU

A chamada precisa apenas executar uma ação?

Escolha um procedimento void.

mostrarMedia(media);
  • exibe uma mensagem;
  • organiza uma ação específica;
  • não fornece um valor para atribuição.

Como os módulos cooperam

1

O main lê

Os dados de entrada são obtidos no fluxo principal.

scanf(...)
2

A função calcula

Recebe os dados, executa a operação e devolve um valor.

calcularMedia(...)
3

O valor retorna

O resultado é armazenado para ser usado pelo programa.

media = ...
4

O procedimento age

Recebe o resultado e realiza uma ação específica.

mostrarMedia(media)
5

O fluxo continua

Depois da chamada, o controle volta ao ponto seguinte do main.

return 0;

Mesmo problema, responsabilidades diferentes

media.c
#include <stdio.h>

float calcularMedia(float a, float b) {
    return (a + b) / 2.0;
}

void mostrarMedia(float media) {
    printf("Media: %.1f\n", media);
}

int main() {
    float n1 = 8.0;
    float n2 = 6.0;
    float media;

    media = calcularMedia(n1, n2);
    mostrarMedia(media);

    return 0;
}

1. A função recebe

Os argumentos 8.0 e 6.0 ocupam os parâmetros a e b.

a = 8.0 e b = 6.0

2. A função devolve

O cálculo produz 7.0. Esse valor volta para a expressão de atribuição.

media = 7.0

3. O procedimento executa

mostrarMedia recebe 7.0 e imprime a mensagem. A chamada não produz um valor para ser armazenado.

Saída: Media: 7.0

Exemplos progressivos

Calcular o dobro

A tarefa produz um número que será usado depois.

  • Entrada do módulo: um número inteiro
  • Processamento: multiplicar por 2
  • Retorno: o valor calculado
Nível básico
dobro.c
#include <stdio.h>

int dobro(int numero) {
    return numero * 2;
}

int main() {
    int resultado;

    resultado = dobro(7);
    printf("%d\n", resultado);

    return 0;
}

Mostrar uma linha de separação

A tarefa apenas produz uma saída visual.

  • Entrada do módulo: quantidade de caracteres
  • Ação: imprimir vários sinais de igual
  • Retorno: nenhum valor
Nível intermediário
linha.c
#include <stdio.h>

void mostrarLinha(int quantidade) {
    for (int i = 0; i < quantidade; i++) {
        printf("=");
    }
    printf("\n");
}

int main() {
    mostrarLinha(12);
    printf("RELATORIO\n");
    mostrarLinha(12);

    return 0;
}

Encontrar e informar o maior valor

Cada módulo recebe uma responsabilidade diferente.

  • A função calcula e devolve o maior dos três números
  • O procedimento recebe o resultado e o apresenta
  • A leitura permanece no main
Nível intermediário
maior.c
#include <stdio.h>

int maiorDeTres(int a, int b, int c) {
    int maior = a;

    if (b > maior) {
        maior = b;
    }
    if (c > maior) {
        maior = c;
    }

    return maior;
}

void informarMaior(int valor) {
    printf("Maior valor: %d\n", valor);
}

int main() {
    int x, y, z, maior;

    scanf("%d %d %d", &x, &y, &z);
    maior = maiorDeTres(x, y, z);
    informarMaior(maior);

    return 0;
}

Erros comuns

Tentar armazenar uma chamada void

Por que acontece: o aluno espera que todo módulo produza um valor.

Como evitar: use uma função com retorno quando a chamada precisar ficar à direita de uma atribuição.

Esquecer ou incompatibilizar o return

Por que acontece: a função declara um tipo de retorno, mas não devolve um valor compatível.

Como evitar: confira se o tipo declarado corresponde ao valor retornado.

Misturar cálculo e apresentação

Por que acontece: um único módulo calcula, lê dados e mostra mensagens ao mesmo tempo.

Como evitar: atribua uma responsabilidade clara a cada função ou procedimento.

Perguntas rápidas

Qual estrutura usar quando o resultado precisa ser armazenado em uma variável?

Use uma função com tipo de retorno compatível, pois a chamada precisa produzir um valor.

Um procedimento void pode receber parâmetros?

Sim. Ele pode receber dados por parâmetros para executar sua ação, embora não devolva um valor por meio de return.

Em C, função e procedimento são estruturas sintaticamente diferentes?

Não. Ambos são funções em C. O termo procedimento é usado didaticamente para uma função cujo tipo de retorno é void.
