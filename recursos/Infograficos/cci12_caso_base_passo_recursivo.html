Ideia central

Recursão é resolver um problema por uma versão menor dele

Uma função recursiva chama a si mesma, mas não pode repetir a mesma situação indefinidamente. Ela precisa de um caso base que encerre as chamadas e de um passo recursivo que aproxime a entrada desse caso.

A pergunta principal é: cada nova chamada está mais perto da saída?
1

Reconheça a saída

Defina a menor situação que pode ser respondida diretamente.

2

Reduza o problema

Faça a nova chamada receber uma entrada menor ou mais simples.

3

Use o retorno

Combine a resposta menor para formar a resposta do problema atual.

Caso base

Responde diretamente à menor entrada e encerra novas chamadas.

n <= 1

Passo recursivo

Chama a função novamente para resolver uma versão menor.

fatorial(n - 1)

Redução do problema

A entrada precisa mudar na direção do caso base.

4 → 3 → 2 → 1

Retorno das chamadas

As respostas voltam em ordem inversa e formam o resultado final.

1 → 2 → 6 → 24

Caso base

A resposta que não precisa de outra chamada.

×
O que faz

Interrompe a recursão quando a entrada já representa o menor problema.

No fatorial

Para n = 1, o resultado é conhecido: 1! = 1.

Trecho: if (n <= 1) return 1;. Sem essa saída, a função continuaria criando chamadas.

Passo recursivo

A regra que reutiliza a própria função.

×
O que faz

Expressa a solução atual usando a resposta de uma entrada menor.

No fatorial

4! pode ser escrito como 4 × 3!.

Trecho: return n * fatorial(n - 1);. A chamada menor ainda precisa retornar para concluir a multiplicação.

Redução do problema

A condição para o caso base ser alcançável.

×
Direção correta

Se a saída ocorre em n = 1, cada chamada deve aproximar n de 1.

Risco

Usar n + 1 faria a entrada se afastar do caso base.

Sequência segura: 4 → 3 → 2 → 1. A redução prova que a recursão tem caminho para terminar.

Retorno das chamadas

A construção da resposta após alcançar a saída.

×
Ordem

A última chamada criada é a primeira a terminar.

No fatorial

Os produtos pendentes são resolvidos de dentro para fora.

Retorno: f(1)=1, f(2)=2, f(3)=6 e f(4)=24.

Anatomia de uma função recursiva

fatorial.c
int fatorial(int n) {
    if (n <= 1) {
        return 1; // caso base
    }
 
    return n * fatorial(n - 1);
}
1

Parâmetro atual

n representa o problema que esta chamada precisa resolver.

2

Teste da saída

Antes de criar outra chamada, a função verifica se chegou ao caso base.

3

Resposta direta

O caso base retorna um valor sem chamar novamente a função.

4

Problema menor

n - 1 aproxima a próxima chamada da saída.

Exemplos progressivos

Básico

Exibir uma contagem até chegar a zero

A função imprime o valor atual e reduz n em uma unidade.

Entrada

contagem(3)

Caso base

Quando n == 0, mostrar “Fim” e retornar.

Passo recursivo

Chamar contagem(n - 1).

Saída

3 2 1 Fim

contagem.c
#include <stdio.h>
 
void contagem(int n) {
    if (n == 0) {
        printf("Fim!\n");
        return;
    }
 
    printf("%d\n", n);
    contagem(n - 1);
}
 
int main(void) {
    contagem(3);
    return 0;
}
Intermediário

Somar os números de 1 até n

Cada chamada soma o valor atual ao resultado da entrada anterior.

Entrada

somaAte(4)

Caso base

Para n <= 0, retornar 0.

Passo recursivo

n + somaAte(n - 1)

Resultado

4 + 3 + 2 + 1 = 10

soma.c
#include <stdio.h>
 
int somaAte(int n) {
    if (n <= 0) {
        return 0;
    }
 
    return n + somaAte(n - 1);
}
 
int main(void) {
    int resultado = somaAte(4);
    printf("Soma: %d\n", resultado);
    return 0;
}
Intermediário

Calcular uma potência inteira

O expoente diminui até zero, quando a resposta conhecida é 1.

Entrada

potencia(2, 4)

Caso base

Para expoente == 0, retornar 1.

Passo recursivo

base * potencia(base, expoente - 1)

Resultado

2 × 2 × 2 × 2 = 16

potencia.c
#include <stdio.h>
 
int potencia(int base, int expoente) {
    if (expoente == 0) {
        return 1;
    }
 
    return base * potencia(base, expoente - 1);
}
 
int main(void) {
    printf("Resultado: %d\n", potencia(2, 4));
    return 0;
}

Erros comuns

Não escrever caso base

Por que acontece: o aluno foca apenas na chamada da própria função.

Como evitar: defina primeiro a menor entrada e sua resposta direta.

Afastar-se da saída

Por que acontece: a próxima chamada aumenta o problema em vez de reduzi-lo.

Como evitar: simule a sequência dos parâmetros até a base.

Retornar valor base incorreto

Por que acontece: o caso base termina, mas entrega uma identidade inadequada.

Como evitar: verifique matematicamente a menor situação.

Ignorar o retorno recursivo

Por que acontece: a função é chamada, mas seu resultado não participa da resposta atual.

Como evitar: combine a chamada em um return quando houver cálculo.

Perguntas rápidas

O que impede uma função recursiva de chamar a si mesma para sempre?

Um caso base alcançável, combinado com uma mudança de parâmetro que aproxima cada chamada dessa condição.

Em fatorial(n - 1), qual é o problema menor?

Calcular o fatorial do número imediatamente anterior. A entrada diminui até chegar a 1.

Quando as multiplicações de fatorial(4) são concluídas?

Depois que o caso base retorna. As chamadas pendentes são resolvidas em ordem inversa: 2, 6 e 24.
