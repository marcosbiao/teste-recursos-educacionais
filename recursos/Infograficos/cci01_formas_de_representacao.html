Ideia central

A solução pode ser a mesma, mesmo quando sua aparência muda

Antes de escrever em C, você pode descrever um algoritmo com frases, comandos estruturados ou símbolos gráficos. O que deve permanecer é a lógica da solução: os mesmos dados, decisões, repetições e resultados.

Escolha a representação que torne o raciocínio mais fácil de compreender, verificar e comunicar.

Exemplo implementado em C

A lógica passa a obedecer às regras sintáticas necessárias para compilação e execução.

Mesmo problema
Problema usado nas quatro formas: ler duas notas, calcular a média e informar se o estudante foi aprovado. A aprovação ocorre quando a média é maior ou igual a 7.
#include <stdio.h>

int main() {
    float nota1, nota2, media;

    printf("Digite as duas notas: ");
    scanf("%f %f", &nota1, &nota2);

    media = (nota1 + nota2) / 2.0;
    printf("Media: %.2f\n", media);

    if (media >= 7.0) {
        printf("Aprovado\n");
    } else {
        printf("Nao aprovado\n");
    }

    return 0;
}

Três formas de pensar antes da implementação

Linguagem natural

Descreve o algoritmo como uma sequência de instruções escritas. É acessível, mas pode gerar ambiguidades se as frases forem vagas.

Mais útil quando: você está entendendo o problema ou explicando a ideia geral a outra pessoa.

Pseudocódigo

Usa comandos padronizados para explicitar ações, condições e repetições. Fica próximo da programação, sem exigir a sintaxe exata de C.

Mais útil quando: você precisa detalhar a lógica e preparar a implementação.

Fluxograma

Organiza a solução por símbolos e setas. Facilita a percepção de caminhos alternativos, ciclos e ordem de execução.

Mais útil quando: você quer visualizar decisões, retornos e relações entre etapas.

Como escolher a representação

1

Precisa explicar a ideia?

Comece com linguagem natural e use frases curtas, objetivas e ordenadas.

2

Precisa detalhar comandos?

Use pseudocódigo para tornar explícitos dados, operações, condições e repetições.

3

Precisa enxergar caminhos?

Use fluxograma quando decisões ou laços dificultarem a compreensão da sequência.

O programa recebe base e altura, calcula a área e apresenta o resultado.

Linguagem natural

Leia a base e a altura. Multiplique os dois valores. Mostre a área calculada.

mostrar area

Implementação em C

A mesma sequência, agora com sintaxe executável
#include <stdio.h>

int main() {
    float base, altura, area;

    scanf("%f %f", &base, &altura);
    area = base * altura;
    printf("Area: %.2f\n", area);

    return 0;
}
Nível intermediário

Identificar o maior de dois valores

O programa recebe dois números e mostra qual deles é maior ou informa que são iguais.

Linguagem natural

Leia dois valores. Compare o primeiro com o segundo. Mostre o maior. Caso sejam iguais, informe a igualdade.

Pseudocódigo

ler a, b
se a > b
    mostrar a
senão se b > a
    mostrar b
senão
    mostrar "valores iguais"

Fluxograma

Início
↓
Ler a e b
↓
a > b?
Sim
Mostrar a
Não
Comparar b > a
O segundo teste separa “b é maior” de “os valores são iguais”.

Implementação em C

A decisão do fluxograma vira uma estrutura condicional
#include <stdio.h>

int main() {
    int a, b;

    scanf("%d %d", &a, &b);

    if (a > b)
        printf("Maior: %d\n", a);
    else if (b > a)
        printf("Maior: %d\n", b);
    else
        printf("Valores iguais\n");

    return 0;
}
Nível intermediário

Somar os números de 1 até N

O programa recebe um número inteiro positivo e calcula a soma de todos os valores de 1 até esse limite.

Linguagem natural

Leia N. Comece a soma em zero. Percorra os números de 1 até N, adicionando cada valor à soma. Mostre o resultado.

Pseudocódigo

ler n
soma ← 0
para i ← 1 até n
    soma ← soma + i
mostrar soma

Fluxograma

Início
↓
Ler N
↓
soma = 0 e i = 1
↓
i ≤ N?
Sim
soma = soma + i
i = i + 1
Não
Mostrar soma
O caminho “Sim” retorna ao teste até que i ultrapasse N.

Implementação em C

O retorno no fluxograma vira uma estrutura de repetição
#include <stdio.h>

int main() {
    int n, i, soma = 0;

    scanf("%d", &n);

    for (i = 1; i <= n; i++) {
        soma = soma + i;
    }

    printf("Soma: %d\n", soma);
    return 0;
}

Erros comuns

Escrever frases vagas

Por que acontece: expressões como “faça o cálculo” não informam quais valores ou operações devem ser usados.

Como evitar: nomeie os dados e descreva cada transformação de forma verificável.

Transformar pseudocódigo em código incompleto

Por que acontece: o aluno mistura comandos genéricos com detalhes específicos de C.

Como evitar: mantenha o pseudocódigo independente da sintaxe, mas logicamente preciso.

Usar símbolos sem conexão

Por que acontece: o fluxograma contém caixas corretas, mas as setas não mostram todos os caminhos.

Como evitar: siga cada saída de decisão e confirme onde o fluxo continua ou retorna.

Perguntas rápidas

Existe uma única representação correta para todo algoritmo?

Não. Diferentes representações podem descrever corretamente a mesma lógica. A escolha depende do objetivo, da complexidade e de quem precisa compreender a solução.

O pseudocódigo precisa seguir exatamente a sintaxe da linguagem C?

Não. Ele deve ser claro e estruturado, mas permanece independente da sintaxe específica de uma linguagem de programação.

Quando o fluxograma costuma trazer mais vantagem?

Quando o algoritmo possui decisões, caminhos alternativos ou repetições que são mais fáceis de perceber visualmente.

Fechamento pedagógico

Depois de estudar este infográfico, o aluno deve conseguir escolher e produzir uma representação adequada para um algoritmo, comparar linguagem natural, pseudocódigo e fluxograma e verificar se a lógica permanece coerente antes da implementação em C.
