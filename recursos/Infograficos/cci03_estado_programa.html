Ideia central

Estado é uma fotografia da memória em um instante

Durante a execução, cada variável possui um valor atual. Quando uma atribuição é executada, o programa calcula o lado direito e substitui o valor armazenado no lado esquerdo. Para saber o resultado, acompanhe as linhas na ordem em que são executadas.

1

Leia o valor atual

Use o estado existente antes de executar a linha.

2

Calcule a expressão

Resolva primeiro tudo o que está à direita do sinal de atribuição.

3

Atualize a memória

Registre o novo valor e use-o nas próximas linhas.

Registrar cada estado

Uma linha por comando mostra a evolução da memória.

Por que registrar?

Quando há várias atualizações, tentar guardar tudo mentalmente aumenta a chance de usar um valor antigo. Registre o estado logo após cada linha executada.

  • saldo = 20 → 20
  • saldo = saldo + 5 → 25
  • saldo = saldo * 2 → 50

Exemplos progressivos

Nível básico

Atualizações em sequência

O mesmo espaço de memória é usado em três momentos. Cada linha parte do valor produzido pela linha anterior.

x = 4x = 4
x = x + 3x = 7
x = x * 2x = 14

Valor exibido: 14

exemplo_1.c
#include <stdio.h>

int main() {
    int x = 4;

    x = x + 3;
    x = x * 2;

    printf("%d\n", x);
    return 0;
}
Nível intermediário

Uma variável depende da outra

Ao atualizar a, o novo valor passa a ser usado na atualização seguinte de b.

a = 5; b = 2a=5 · b=2
a = a + ba=7 · b=2
b = a - ba=7 · b=5
a++a=8 · b=5

Estado final: a = 8 e b = 5

exemplo_2.c
#include <stdio.h>

int main() {
    int a = 5;
    int b = 2;

    a = a + b;
    b = a - b;
    a++;

    printf("a = %d, b = %d\n", a, b);
    return 0;
}
Nível intermediário

O caminho executado altera o estado

A condição decide se uma atualização ocorrerá. Somente as linhas realmente executadas entram no rastreamento.

saldo = 12; bonus = 312 · 3
saldo += bonussaldo = 15
saldo *= 2saldo = 30
saldo -= 4saldo = 26

Como 15 ≥ 15, a atualização dentro do if é executada.

exemplo_3.c
#include <stdio.h>

int main() {
    int saldo = 12;
    int bonus = 3;

    saldo += bonus;

    if (saldo >= 15) {
        saldo *= 2;
    }

    saldo -= 4;
    printf("Saldo: %d\n", saldo);

    return 0;
}

Erros comuns

Ler atribuição como equação

Por que acontece: em matemática, x = x + 1 parece impossível.

Como evitar: leia como “calcule o valor atual de x mais 1 e armazene novamente em x”.

Continuar usando o valor antigo

Por que acontece: o aluno calcula uma atualização, mas não substitui o estado anterior.

Como evitar: registre o novo valor imediatamente após cada atribuição.

Pular linhas ou caminhos

Por que acontece: o rastreamento ignora a ordem de execução ou uma condição.

Como evitar: acompanhe somente as linhas executadas e respeite a sequência do programa.

Perguntas rápidas

Se x vale 6 e o programa executa x = x + 4, qual será o novo estado?

O lado direito calcula 6 + 4. Depois, o resultado 10 substitui o valor anterior. O novo estado é x = 10.

Em x = x * 2, qual uso de x é lido primeiro?

O x do lado direito é lido primeiro para calcular a expressão. O resultado é então escrito no x do lado esquerdo.

Uma atribuição altera os valores que a variável teve no passado?

Não. Ela cria um novo estado a partir daquele ponto da execução. Os valores anteriores servem apenas para compreender o histórico do programa.

Fechamento pedagógico

Depois de estudar este infográfico, o aluno deve conseguir executar mentalmente comandos simples em C, registrar os valores atuais das variáveis após cada linha e determinar corretamente o estado final do programa.
