Ideia central

Leia o problema antes de escolher a estrutura

Uma estrutura é adequada quando corresponde à ação principal exigida pelo problema. Decidir entre caminhos sugere condição. Repetir uma ação sugere laço. Guardar vários valores pode exigir vetor ou matriz. Isolar uma tarefa com nome sugere função.

A escolha não é exclusiva. Um vetor costuma ser percorrido por um for, e uma condição pode aparecer dentro desse laço.

1

2

3

Combine com propósito

Adicione outras estruturas somente quando elas cumprirem uma função clara.

Combinações frequentes

Vetor + repetição

O vetor armazena os valores e o laço visita cada posição.

for (...) soma += notas[i];

Repetição + condição

O laço percorre os dados e a condição seleciona alguns deles.

if (valor % 2 == 0)

Função + estruturas

A função pode conter condição, repetição, vetor ou matriz.

float media(float v[], int n)

Matriz + laços aninhados

Um laço controla as linhas e outro percorre as colunas.

for (i...) for (j...)

Um problema, várias estruturas

Problema: registrar as notas de duas turmas com três estudantes, calcular a média de cada turma e informar quantos estudantes foram aprovados.
matriz

Armazena as notas por turma e estudante.

for

Percorre linhas e colunas.

if

Conta notas maiores ou iguais a 7.

função

Calcula a média de uma turma.

combinação_estruturas.c
#include <stdio.h>

float mediaTurma(float notas[], int quantidade) {
    float soma = 0;

    for (int i = 0; i < quantidade; i++) {
        soma += notas[i];
    }

    return soma / quantidade;
}

int main() {
    float notas[2][3] = {
        {8.0, 6.0, 9.0},
        {5.0, 7.5, 7.0}
    };

    for (int t = 0; t < 2; t++) {
        int aprovados = 0;

        for (int a = 0; a < 3; a++) {
            if (notas[t][a] >= 7.0) {
                aprovados++;
            }
        }

        printf("Turma %d: media %.1f, aprovados %d\n",
               t + 1, mediaTurma(notas[t], 3), aprovados);
    }

    return 0;
}

Decidir entre dois caminhos

Leia uma idade e informe se a pessoa é maior de idade.

  • Existe uma única entrada.
  • Não há repetição nem coleção de dados.
  • A resposta depende de um teste relacional.

Leia cinco notas, armazene-as e calcule a média.

  • Há vários valores do mesmo tipo.
  • Todos precisam ser guardados e percorridos.
  • O cálculo da média pode ser separado em função.

#include <stdio.h>

float calcularMedia(float notas[], int n) {
    float soma = 0;

    for (int i = 0; i < n; i++) {
        soma += notas[i];
    }

    return soma / n;
}

int main() {
    float notas[5];

    for (int i = 0; i < 5; i++) {
        scanf("%f", &notas[i]);
    }

    printf("Media: %.2f\n", calcularMedia(notas, 5));
    return 0;
}

Erros comuns

Escolher por uma palavra

Por que acontece: o aluno encontra “cada” e decide usar vetor, mesmo sem precisar guardar os valores.

Como evitar: identifique a ação e verifique se o armazenamento é realmente necessário.

Usar vetor sem necessidade

Por que acontece: vários dados são lidos, mas apenas soma ou contagem precisa ser mantida.

Como evitar: use vetor somente quando os valores precisarem ser acessados depois.

Usar matriz para uma lista

Por que acontece: o aluno confunde quantidade de valores com quantidade de dimensões.

Como evitar: escolha matriz apenas quando linha e coluna tiverem significado.

Procurar uma única estrutura

Por que acontece: o aluno pensa que condição, laço e função são alternativas excludentes.

Como evitar: defina uma estrutura principal e combine as demais por responsabilidade.

Perguntas rápidas

Um problema pede a soma de 100 valores, mas não precisa mostrar os valores depois. É obrigatório usar vetor?

Não. Um laço e um acumulador podem resolver o problema sem armazenar todos os valores. O vetor só é necessário quando os valores precisam permanecer disponíveis.

Uma matriz substitui os laços aninhados?

Não. A matriz organiza os dados em duas dimensões. Para percorrer todas as linhas e colunas, normalmente são usados dois laços aninhados.

Uma função pode conter condição e repetição?

Sim. A função organiza uma tarefa. Dentro dela podem existir condições, laços, vetores e matrizes, desde que contribuam para a responsabilidade daquela função.

Use condição para escolher caminhos

A execução depende do resultado verdadeiro ou falso de um teste.

Exemplo

Informar se uma nota é suficiente para aprovação.

Pista no enunciado“Se a nota for maior ou igual a 7...”
Estruturaif (nota >= 7)

Cuidado: uma condição escolhe caminhos. Ela não repete automaticamente o teste para vários dados.

Use repetição para executar uma ação várias vezes

O laço controla quando começar, continuar, atualizar e parar.

Exemplo

Pista no enunciado“Leia cinco valores”
Estruturafor (i = 0; i < 5; i++)

Cuidado: repetir não significa armazenar. Se os valores não forem usados depois, talvez um vetor não seja necessário.
