Ideia central

O laço repete comandos, mas as variáveis guardam a história da repetição

A cada passagem, algumas variáveis contam eventos, outras reúnem resultados e outra variável ou expressão decide se o processo continua.

Papéis diferentes podem ser exercidos por variáveis diferentes ou pela mesma variável.
1

Contar

Registra quantas repetições, ocorrências ou valores já foram processados.

2

Acumular

Combina o valor atual com os valores obtidos nas repetições anteriores.

3

Controlar

Participa da condição que permite continuar ou encerrar o laço.

Como reconhecer cada papel

Contador

O valor muda em passos regulares?

Geralmente começa em 0 ou 1 e recebe incrementos como 1 a cada ocorrência.

quantidade++;

Acumulador

O valor novo depende do valor anterior?

Recebe o resultado já acumulado combinado com o dado da repetição atual.

soma += numero;

Controle

O valor aparece na condição do laço?

Quando muda, pode manter a repetição ativa ou produzir a condição de parada.

numero != 0

Acompanhe o estado do laço

Os papéis podem se sobrepor

Uma variável pode cumprir mais de uma função

No laço abaixo, i conta as repetições e também controla a continuidade. Não é necessário criar duas variáveis diferentes quando um único estado representa os dois papéis.

Pense no papel, não no nome

Nomes como i, cont ou total ajudam, mas o papel é determinado pelo uso da variável.

Exemplos progressivos

Nível básico

Somar os números de 1 a 5

O número de repetições é conhecido. A mesma variável i funciona como contador e controle, enquanto soma acumula os valores.

Contador e controle: i
Acumulador: soma
Estado final: i = 6 e soma = 15
soma_1_a_5.c
#include <stdio.h>

int main() {
    int soma = 0;

    for (int i = 1; i <= 5; i++) {
        soma += i;
    }

    printf("Soma: %d\n", soma);
    return 0;
}
Nível intermediário

Calcular a média de cinco notas

O contador controla as cinco leituras, o acumulador soma as notas e a variável nota guarda o dado atual.

Contador e controle: i
Acumulador: soma
Dado temporário: nota, que não controla o laço
media_notas.c
#include <stdio.h>

int main() {
    float nota, soma = 0.0;

    for (int i = 0; i < 5; i++) {
        scanf("%f", &nota);
        soma += nota;
    }

    float media = soma / 5;
    printf("Media: %.2f\n", media);
    return 0;
}
Nível intermediário

Ler valores até receber zero

Agora os três papéis aparecem em variáveis distintas. O zero é uma sentinela: indica a parada e não é processado.

Contador: quantidade
Acumulador: soma
Controle: numero na condição numero != 0
sentinela.c
#include <stdio.h>

int main() {
    int numero;
    int quantidade = 0;
    int soma = 0;

    scanf("%d", &numero);

    while (numero != 0) {
        quantidade++;
        soma += numero;
        scanf("%d", &numero);
    }

    printf("Quantidade: %d\n", quantidade);
    printf("Soma: %d\n", soma);
    return 0;
}

Erros comuns

Não inicializar o acumulador

Por que acontece: o aluno começa a somar sobre um valor indefinido.

Como evitar: use o elemento neutro adequado, como 0 para soma e 1 para produto.

Somar no contador

Por que acontece: contador e acumulador são tratados como se tivessem a mesma atualização.

Como evitar: contador recebe um passo regular; acumulador recebe o dado atual.

Não atualizar o controle

Por que acontece: a condição continua verdadeira para sempre.

Como evitar: identifique qual comando muda o valor relacionado à parada em cada repetição.

Perguntas rápidas

Um contador precisa sempre começar em zero?

Não. Ele deve começar no valor coerente com o que representa. Para contar ocorrências processadas, 0 é comum. Para percorrer uma sequência que começa em 1, pode iniciar em 1.

Por que um acumulador de soma costuma começar em zero?

Porque zero é o elemento neutro da adição. Somar zero ao primeiro valor não altera esse valor.

A variável de um for pode ser contador e controle ao mesmo tempo?

Sim. Em for (i = 0; i < 5; i++), i registra o avanço da contagem e participa da condição que encerra o laço.

Registra ocorrências

Como funciona o contador

×
No exemplo 4, 7 e 0: quantidade começa em 0 e aumenta somente quando um valor diferente de zero é processado. Ao final, vale 2.
Inicialização

Use o valor que representa nenhuma ocorrência contada, normalmente 0.

Atualização

O passo costuma ser fixo, como quantidade++.

Pergunta respondida

Quantos elementos, acertos ou repetições ocorreram?

int quantidade = 0;

// numero vale 4
quantidade++; // 1

// numero vale 7
quantidade++; // 2

Reúne resultados

Como funciona o acumulador

×
No exemplo 4, 7 e 0: soma começa em 0, recebe 4 e depois recebe mais 7. O estado final é 11.
Inicialização

Para soma, use 0. Para produto, use 1.

Atualização

O novo valor depende do acumulado anterior e do dado atual.

Pergunta respondida

Qual total, produto ou resultado parcial foi construído?

acumulador.c
int soma = 0;

// numero vale 4
soma = soma + 4; // 4

// numero vale 7
soma += 7; // 11

// numero vale 0: não acumula
Decide a continuidade

Como funciona a variável de controle

×
No exemplo 4, 7 e 0: numero participa da condição numero != 0. Quando recebe 0, a condição fica falsa e nenhuma nova repetição começa.
Antes do laço

Ela precisa ter um valor definido para que a primeira condição possa ser avaliada.

Dentro do laço

Seu estado deve ser atualizado para que a parada possa ser alcançada.

Formas comuns

Contador, sentinela, opção de menu ou indicador lógico.

controle.c
scanf("%d", &numero);

while (numero != 0) {
    // processa 4 e depois 7

    // atualiza a variável de controle
    scanf("%d", &numero);
}

// ao receber 0, o laço termina
