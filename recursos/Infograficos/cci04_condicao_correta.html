Ideia central

Uma condição é uma pergunta feita ao programa

Ela compara valores e produz apenas um de dois resultados: verdadeiro ou falso. Esse resultado determina qual caminho do programa será executado.

idade >= 18 → verdadeiro ou falso
1

Compare

Use operadores relacionais para perguntar como dois valores se relacionam.

2

Combine

Use && ou || quando uma decisão depende de mais de uma comparação.

3

Negue

Use ! para inverter um resultado lógico já existente.

Operadores relacionais comparam dois valores

Considere um sistema que verifica se uma pessoa pode entrar em um evento. A idade informada será comparada com o limite mínimo.

  • idade >= 18: idade mínima atingida
  • idade == 18: idade exatamente igual a 18
  • idade != 18: idade diferente de 18

Teste com valores

17 >= 18falso
18 >= 18verdadeiro
20 != 18verdadeiro

&& exige que todas as regras sejam atendidas

No mesmo evento, a entrada exige idade mínima e ingresso válido. As duas comparações precisam ser verdadeiras.

  • Regra 1: idade >= 18
  • Regra 2: temIngresso == 1
  • Condição: idade >= 18 && temIngresso == 1

Quando libera?

V && Vverdadeiro
V && Ffalso
F && Vfalso

|| permite caminhos alternativos

Uma credencial pode ser validada por ingresso digital ou por nome na lista. Basta uma das alternativas funcionar.

  • Alternativa 1: ingressoDigital == 1
  • Alternativa 2: estaNaLista == 1
  • Condição: ingressoDigital == 1 || estaNaLista == 1

Quando libera?

F || Vverdadeiro
V || Fverdadeiro
F || Ffalso

! inverte uma condição

Se a variável bloqueado vale 1 quando o acesso está impedido, !bloqueado representa a ideia de “não está bloqueado”.

  • bloqueado = 1 → verdadeiro
  • !bloqueado → falso
  • bloqueado = 0 → !bloqueado é verdadeiro

Inversão lógica

!verdadeirofalso
!falsoverdadeiro
!0verdadeiro

Como montar uma condição correta

1

Escreva a regra em palavras

Exemplo: “a nota deve estar entre 0 e 10”.

2

Separe as comparações

nota >= 0 e nota <= 10.

3

Escolha o conector

As duas precisam valer, então use &&.

4

Teste valores simples

Experimente um valor válido, um abaixo e outro acima do limite.

Exemplos progressivos

Exemplo 1: verificar maioridade

O programa lê uma idade e informa se a pessoa tem 18 anos ou mais.

Construção da condição

Regra em palavrasA idade precisa ser maior ou igual a 18.
Condição em Cidade >= 18
Por que >=O valor 18 deve ser aceito, não apenas valores maiores.
Entrada17 → falso
Limite18 → verdadeiro
Acima25 → verdadeiro
maioridade.c
#include <stdio.h>

int main() {
    int idade;

    scanf("%d", &idade);

    if (idade >= 18) {
        printf("Maior de idade");
    } else {
        printf("Menor de idade");
    }

    return 0;
}

Exemplo 2: validar uma nota

Uma nota é válida somente quando está entre 0 e 10, incluindo os dois limites.

Duas regras precisam valer

Limite inferiornota >= 0
Limite superiornota <= 10
Combinação corretanota >= 0 && nota <= 10
Abaixo-1 → falso
Dentro7.5 → verdadeiro
Acima11 → falso
validar_nota.c
#include <stdio.h>

int main() {
    float nota;

    scanf("%f", &nota);

    if (nota >= 0 && nota <= 10) {
        printf("Nota valida");
    } else {
        printf("Nota invalida");
    }

    return 0;
}

Exemplo 3: liberar acesso por senha ou biometria

O acesso é permitido quando uma credencial é válida e o usuário não está bloqueado.

Agrupamento da lógica

Alternativas de autenticaçãosenhaCorreta || biometriaValida
Restrição!bloqueado
Condição completa(senhaCorreta || biometriaValida) && !bloqueado
Senha válidalibera
Biometria válidalibera
Bloqueadonão libera
acesso.c
#include <stdio.h>

int main() {
    int senhaCorreta, biometriaValida, bloqueado;

    scanf("%d %d %d", &senhaCorreta,
          &biometriaValida, &bloqueado);

    if ((senhaCorreta || biometriaValida)
        && !bloqueado) {
        printf("Acesso liberado");
    } else {
        printf("Acesso negado");
    }

    return 0;
}

Erros comuns

Usar = no lugar de ==

Por que acontece: o símbolo de atribuição é confundido com comparação de igualdade.

Como evitar: leia == como “é igual a” dentro da condição.

if (x = 5)if (x == 5)

Escrever uma faixa como na matemática

Por que acontece: expressões encadeadas como 0 <= nota <= 10 parecem naturais, mas não representam a faixa corretamente em C.

Como evitar: escreva duas comparações e conecte com &&.

0 <= nota <= 10nota >= 0 && nota <= 10

Trocar E por OU

Por que acontece: o aluno escolhe o operador pelo som da frase, sem verificar quais casos devem ser aceitos.

Como evitar: use && quando todas as regras forem obrigatórias e || quando houver alternativas.

Omitir parênteses em condições longas

Por que acontece: o agrupamento pretendido fica apenas na cabeça de quem escreveu.

Como evitar: agrupe partes relacionadas, como (senha || biometria) && !bloqueado.

Perguntas rápidas

Qual operador verifica se dois valores são iguais?

O operador de igualdade é ==. Um único = realiza atribuição.

Para aceitar uma nota apenas entre 0 e 10, deve-se usar E ou OU?

Use &&, porque a nota precisa atender simultaneamente ao limite inferior e ao limite superior.

O que acontece quando aplicamos ! a uma condição verdadeira?

O resultado é invertido e passa a ser falso.

Fechamento pedagógico

Depois de estudar este infográfico, o aluno deve conseguir traduzir uma regra em comparações, escolher entre &&, || e !, testar valores simples e justificar por que a condição produz o resultado esperado.
