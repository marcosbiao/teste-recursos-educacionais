Ideia central

Uma cadeia de condições divide o conjunto de entradas em regiões

Uma cobertura adequada ocorre quando cada entrada válida encontra um caminho e, quando os casos são exclusivos, não pertence a mais de uma faixa. Os valores próximos às mudanças de faixa são os que mais revelam erros.

A fronteira deve pertencer a uma faixa definida

Considere que notas de 0 a 59 são reprovadas e notas de 60 a 69 ficam em recuperação. O valor 60 é a fronteira.

  • Teste imediatamente antes: 59.
  • Teste a própria fronteira: 60.
  • Teste imediatamente depois: 61.

Faixas coerentes

if (nota < 60) {
    printf("Reprovado");
} else if (nota < 70) {
    printf("Recuperacao");
}
×

Uma lacuna deixa uma entrada sem caminho

As condições abaixo não tratam a nota 60. Ela não é menor que 60 e também não é maior que 60.

  • nota < 60
  • nota > 60
  • Valor perdido: nota == 60

Trecho com lacuna

if (nota < 60) {
    printf("Reprovado");
} else if (nota > 60) {
    printf("Outro caso");
}
×

Condições sobrepostas podem classificar duas vezes

Com dois if independentes, uma nota entre 60 e 70 pode satisfazer as duas regras e gerar duas saídas.

  • nota >= 60
  • nota <= 70
  • A nota 65 atende às duas condições.

Trecho sobreposto

if (nota >= 60)
    printf("Passou do limite");

if (nota <= 70)
    printf("Ate 70");
×

Entradas fora do domínio também precisam de resposta

Se a nota válida deve estar entre 0 e 100, valores negativos ou maiores que 100 devem ser reconhecidos como inválidos.

  • Valor abaixo do domínio: -3.
  • Valor acima do domínio: 105.
  • O tratamento deve ocorrer antes das classificações válidas.

Validação inicial

if (nota < 0 ||
    nota > 100) {
    printf("Nota invalida");
}

Como verificar a cobertura

1

Liste as faixas

Escreva cada condição como um intervalo de valores.

2

Marque os limites

Localize os pontos em que a resposta deveria mudar.

3

Teste três valores

Use antes do limite, o limite e depois do limite.

4

Compare as saídas

Confirme se cada valor produz apenas o resultado esperado.

Exemplos progressivos

Exemplo 1: classificar o sinal de um número

Três casos simples cobrem todos os números inteiros sem sobreposição.

Teste de fronteira: −1, 0 e 1
Antes-1 deve ser negativo.
Fronteira0 deve cair no caso zero.
Depois1 deve ser positivo.
Cobertura: qualquer inteiro é menor que zero, igual a zero ou maior que zero.
sinal.c
#include <stdio.h>

int main() {
    int numero;
    scanf("%d", &numero);

    if (numero < 0) {
        printf("Negativo");
    } else if (numero == 0) {
        printf("Zero");
    } else {
        printf("Positivo");
    }

    return 0;
}

Exemplo 2: corrigir a lacuna no valor 500

Uma diferença entre > e >= pode deixar um valor sem classificação.

Valores de teste: 499, 500 e 501
Antes499 entra na faixa intermediária.
Limite500 revela a lacuna se a última condição usar apenas > 500.
Depois501 recebe o maior desconto.
Correção: a última faixa começa em 500, portanto deve usar valor >= 500.
desconto.c
#include <stdio.h>

int main() {
    float valor;
    scanf("%f", &valor);

    if (valor < 100) {
        printf("Sem desconto");
    } else if (valor < 500) {
        printf("Desconto de 5%%");
    } else {
        printf("Desconto de 10%%");
    }

    return 0;
}

Exemplo 3: cobrir faixas válidas e entradas inválidas

A validação do domínio ocorre antes das classificações acadêmicas.

Ordem das condições: primeiro valide o domínio. Depois use limites crescentes para formar faixas contínuas e exclusivas.
classificacao.c
#include <stdio.h>

int main() {
    int nota;
    scanf("%d", &nota);

    if (nota < 0 || nota > 100) {
        printf("Nota invalida");
    } else if (nota < 60) {
        printf("Reprovado");
    } else if (nota < 70) {
        printf("Recuperacao");
    } else {
        printf("Aprovado");
    }

    return 0;
}

Erros comuns

Testar apenas valores comuns

Por que acontece: o aluno usa um valor no meio de cada faixa e não verifica as transições.

Como evitar: teste antes, no limite e depois do limite.

Trocar >= por >

Por que acontece: a igualdade no ponto inicial da faixa é esquecida.

Como evitar: escreva os intervalos no papel antes de programar.

idade > 18idade >= 18

Usar if independentes

Por que acontece: duas regras sobrepostas podem executar no mesmo teste.

Como evitar: use else if quando apenas uma classificação deve ser produzida.

Ignorar entradas inválidas

Por que acontece: o aluno pensa apenas nas faixas esperadas.

Como evitar: defina o domínio e trate valores fora dele explicitamente.

Perguntas rápidas

Quais três valores devem ser testados ao redor da fronteira 18?

Teste 17, 18 e 19. Assim você observa o comportamento antes do limite, no próprio limite e depois dele.

As condições x < 10 e x > 10 cobrem todos os inteiros?

Não. O valor 10 não atende a nenhuma das duas condições e forma uma lacuna.

Por que duas condições verdadeiras podem ser um problema?

Quando os casos deveriam ser exclusivos, uma sobreposição pode produzir duas saídas ou tornar o resultado dependente da ordem das condições.
