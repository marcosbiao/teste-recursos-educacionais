Ideia central

O local da declaração cria uma fronteira de acesso

O escopo indica em quais partes do código um nome pode ser usado. Uma variável declarada dentro de uma função ou de um bloco é local àquela região. Uma variável declarada fora das funções possui escopo de arquivo e costuma ser chamada de global.

Neste infográfico, “duração da variável local” refere-se às variáveis locais comuns, também chamadas de automáticas. Uma variável local declarada com static possui duração diferente.

De fora para dentro

Um bloco interno pode usar nomes declarados em regiões externas que ainda estejam visíveis.

De dentro para fora

O código externo não pode acessar uma variável que nasceu apenas dentro de um bloco interno.

Ciclo de uma variável local comum

1

Entrar no bloco

O fluxo alcança a função ou o bloco que contém a declaração.

if (numero > 0)
2

Declarar

O nome passa a estar disponível a partir do ponto da declaração.

int dobro;
3

Inicializar e usar

A variável recebe um valor e participa dos comandos daquele escopo.

dobro = numero * 2;
4

Sair do bloco

O nome deixa de estar visível e a variável automática encerra sua duração.

}
5

Não acessar depois

Uma tentativa de usar o nome fora do escopo causa erro de compilação.

printf("%d", dobro);

Global ou fluxo explícito?

Dependência global

A função lê ou altera um estado externo sem mostrar isso na assinatura.

int total = 0; void somar(int valor) { total += valor; }
  • o resultado depende de um dado externo;
  • qualquer função com acesso pode alterar o estado;
  • testes e reutilização ficam mais difíceis.
→

Fluxo por parâmetro e retorno

A entrada e a saída ficam explícitas na interface da função.

int somar(int total, int valor) { return total + valor; }
  • os dados necessários aparecem nos parâmetros;
  • o novo valor é produzido pelo retorno;
  • a função fica mais previsível e reutilizável.
Estado compartilhado

Use global apenas quando o dado realmente representar um estado compartilhado do programa e houver justificativa clara.

Constantes do programa

Valores fixos podem ser centralizados, mas devem ser identificados como constantes e protegidos contra alteração indevida.

Regra prática

Quando uma função precisa de um dado, tente recebê-lo por parâmetro. Quando produz um resultado, tente devolvê-lo por retorno.

Exemplos progressivos

Variável criada dentro do if

metade só existe no bloco delimitado pelas chaves da condição.

  • numero pertence ao main;
  • metade pertence apenas ao bloco if;
  • usar metade depois da chave final causaria erro.
Nível básico
bloco_interno.c
#include <stdio.h>

int main() {
    int numero = 8;

    if (numero % 2 == 0) {
        int metade = numero / 2;
        printf("Metade: %d\n", metade);
    }

    // metade não pode ser usada aqui
    return 0;
}

Parâmetros e resultado local

Os parâmetros e a variável media pertencem à função calcularMedia.

  • nota1 e nota2 recebem cópias dos argumentos;
  • media não pode ser usada no main;
  • o resultado atravessa a fronteira por meio de return.
Nível intermediário
media_local.c
#include <stdio.h>

float calcularMedia(float nota1, float nota2) {
    float media = (nota1 + nota2) / 2.0f;
    return media;
}

int main() {
    float resultado = calcularMedia(8.0f, 6.0f);
    printf("Media: %.1f\n", resultado);
    return 0;
}

Dois resultado, duas memórias

O nome é repetido, mas cada declaração pertence a uma função diferente.

  • o resultado de aplicarBonus é local à função;
  • o resultado do main recebe apenas o valor retornado;
  • alterar um deles não altera automaticamente o outro.
Nível intermediário
nomes_iguais.c
#include <stdio.h>

int aplicarBonus(int pontos) {
    int resultado = pontos + 10;
    return resultado;
}

int main() {
    int resultado = aplicarBonus(50);
    printf("Pontos: %d\n", resultado);
    return 0;
}

Erros comuns

Usar uma variável fora do bloco

Por que acontece: o aluno vê a declaração acima e ignora a chave que encerrou o escopo.

Como evitar: trace visualmente o par de chaves que contém a declaração.

Confundir nomes iguais com a mesma variável

Por que acontece: duas funções usam o mesmo identificador.

Como evitar: associe cada declaração ao seu próprio escopo e espaço de memória.

Usar global por conveniência

Por que acontece: parece mais simples acessar o dado de qualquer função.

Como evitar: prefira parâmetros e retornos para tornar o fluxo explícito.

Ler variável local sem inicializar

Por que acontece: declarar uma variável local comum não atribui automaticamente um valor confiável.

Como evitar: inicialize antes da primeira leitura.

Perguntas rápidas

Uma variável declarada dentro de um if pode ser usada depois da chave final?

Não. O nome pertence ao bloco do if e deixa de estar visível quando esse bloco termina.

Um parâmetro é uma variável global?

Não. O parâmetro é local à função e recebe um valor quando a função é chamada.

Variáveis globais são sempre proibidas?

Não. Elas podem representar estado realmente compartilhado, mas devem ser justificadas. Para comunicação comum entre funções, parâmetros e retornos costumam produzir código mais claro e previsível.
