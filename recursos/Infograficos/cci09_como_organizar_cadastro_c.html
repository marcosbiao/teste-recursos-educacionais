Ideia central

Um cadastro é uma coleção de registros

Um registro agrupa campos que descrevem um único elemento. Um cadastro com vários elementos pode ser representado por um vetor desse tipo de registro.

Pense assim: um registro = uma ficha e um vetor de registros = um conjunto de fichas.

1

Defina os campos

Escolha apenas os dados necessários para descrever cada elemento.

2

Crie o tipo

Use struct para manter os campos reunidos sob uma mesma estrutura.

3

Guarde vários

Declare um vetor de registros e percorra suas posições com um laço.

Fluxo para montar um cadastro

1. Identifique

Liste os campos necessários para cada elemento.

2. Defina

Crie a struct com tipos coerentes.

3. Declare

Crie um vetor do tipo do registro.

4. Percorra

Use um índice para ler ou mostrar cada ficha.

5. Acesse

Use vetor[i].campo nas operações.

Exemplos progressivos

Exemplo básico

Leia e liste três produtos

Cada produto possui código, nome e preço. O laço percorre o vetor e mantém os três campos unidos.

  • Um tipo descreve o formato de cada produto.
  • O vetor guarda três registros completos.
  • O acesso usa produtos[i].campo.
structvetorfor

Observe: a posição i seleciona o produto. O ponto seleciona um campo desse produto.

produtos.c
#include <stdio.h>

typedef struct {
    int codigo;
    char nome[30];
    float preco;
} Produto;

int main() {
    Produto produtos[3];

    for (int i = 0; i < 3; i++) {
        scanf("%d", &produtos[i].codigo);
        scanf("%29s", produtos[i].nome);
        scanf("%f", &produtos[i].preco);
    }

    for (int i = 0; i < 3; i++) {
        printf("%d | %s | R$ %.2f\n",
               produtos[i].codigo,
               produtos[i].nome,
               produtos[i].preco);
    }

    return 0;
}
Exemplo intermediário

Busque um livro pelo código

O cadastro guarda código, título e disponibilidade. Um laço procura o registro cujo código corresponde ao valor informado.

  • A variável encontrado registra o resultado da busca.
  • O código identifica o livro.
  • Os demais campos são recuperados do mesmo registro.
structbusca linearif

Vantagem: título e disponibilidade não precisam ser sincronizados manualmente com vetores separados.

livros.c
#include <stdio.h>

typedef struct {
    int codigo;
    char titulo[40];
    int disponivel;
} Livro;

int main() {
    Livro livros[3] = {
        {10, "Algoritmos", 1},
        {20, "Redes", 0},
        {30, "BancoDados", 1}
    };
    int buscado, encontrado = 0;

    scanf("%d", &buscado);

    for (int i = 0; i < 3; i++) {
        if (livros[i].codigo == buscado) {
            printf("%s | %s\n",
                   livros[i].titulo,
                   livros[i].disponivel ? "disponivel" : "emprestado");
            encontrado = 1;
        }
    }

    if (!encontrado) {
        printf("Livro nao encontrado.\n");
    }

    return 0;
}
Exemplo intermediário

Calcule a média e mostre a situação

Cada aluno possui nome e duas notas. O programa percorre os registros, calcula a média de cada aluno e apresenta sua situação.

  • As notas pertencem ao mesmo aluno.
  • A média é calculada a partir do registro atual.
  • Uma função separa o cálculo da apresentação.
structfunçãoforif

Observe: o cadastro organiza os dados; a função organiza uma operação realizada sobre esses dados.

alunos.c
#include <stdio.h>

typedef struct {
    char nome[30];
    float nota1;
    float nota2;
} Aluno;

float calcularMedia(Aluno aluno) {
    return (aluno.nota1 + aluno.nota2) / 2.0;
}

int main() {
    Aluno alunos[3] = {
        {"Ana", 8.0, 9.0},
        {"Bruno", 5.0, 6.0},
        {"Carla", 7.0, 7.5}
    };

    for (int i = 0; i < 3; i++) {
        float media = calcularMedia(alunos[i]);
        printf("%s: %.1f | %s\n",
               alunos[i].nome, media,
               media >= 7.0 ? "aprovado" : "reprovado");
    }

    return 0;
}

Erros comuns

Usar vetores separados

Por que acontece: cria-se um vetor para nomes, outro para códigos e outro para notas.

Como evitar: agrupe os campos de cada elemento em uma struct.

Confundir capacidade e quantidade

Por que acontece: o programa percorre todas as posições, mesmo as não preenchidas.

Como evitar: mantenha uma variável com o número de registros válidos.

Acessar o campo sem posição

Por que acontece: tenta-se usar alunos.nota quando alunos é um vetor.

Como evitar: selecione primeiro o registro: alunos[i].nota.

Ultrapassar o vetor

Por que acontece: novos registros são inseridos sem verificar o limite.

Como evitar: valide se quantidade < capacidade antes de cadastrar.

Perguntas rápidas

Qual é a diferença entre um registro e um vetor de registros?

Um registro representa um único elemento com seus campos. Um vetor de registros armazena vários elementos que seguem essa mesma estrutura.

O que significa alunos[2].nota?

Significa acessar o campo nota do registro armazenado no índice 2 do vetor alunos.

Por que guardar a quantidade de registros cadastrados?

Porque a capacidade do vetor pode ser maior que a quantidade de posições realmente preenchidas. O laço deve percorrer apenas os registros válidos.
