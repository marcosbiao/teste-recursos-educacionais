Como uma função recebe informações

Entenda onde aparecem os parâmetros, quais valores são enviados como argumentos e como uma chamada conecta o fluxo principal ao corpo da função.

Da definição para a execuçãoO parâmetro prepara o lugar. O argumento fornece o valor.
Definição da função int dobro(int numero)

numero é um parâmetro: um nome local que receberá um valor em cada chamada.

→
Chamada da função resultado = dobro(7);

7 é o argumento enviado. A expressão dobro(7) é a chamada.

Relação essencial: o argumento ocupa a posição correspondente ao parâmetro e fornece o valor usado dentro da função.

Ideia central

A definição cria entradas nomeadas para a função

Os parâmetros aparecem na definição e descrevem quais informações a função espera receber. Os argumentos aparecem na chamada e fornecem valores concretos para essas entradas.

Para tipos simples neste nível introdutório, cada parâmetro recebe uma cópia do valor do argumento. O parâmetro é uma variável local da função.
1

Definir

Informe tipo, nome da função e parâmetros necessários.

2

Chamar

Use o nome da função e envie argumentos entre parênteses.

3

Associar

Relacione cada argumento ao parâmetro correspondente por posição.

Mapa da passagem de informações

Clique em cada bloco para abrir uma explicação. Todos usam o mesmo exemplo de cálculo da área de um retângulo.
Uma função recebe dados calcularArea(5.0, 3.0) a chamada envia valores, os parâmetros os recebem e o corpo executa o cálculo.

Parâmetro

Nome local declarado na definição para receber uma informação.

float largura

Argumento

Valor, variável ou expressão enviada quando a função é chamada.

5.0f

Chamada

Expressão que transfere o fluxo para a função usando seu nome.

calcularArea(...)

Correspondência

Quantidade, ordem e tipos devem combinar com a definição.

1º → 1º | 2º → 2º

Anatomia da definição e da chamada

Na definição

Os parâmetros especificam o formato dos dados esperados.

float calcularArea(float largura, float altura)
  • float define o tipo de cada parâmetro;
  • largura e altura são nomes locais;
  • esses nomes podem ser usados dentro do corpo da função.
↔

Na chamada

Os argumentos fornecem valores para os parâmetros.

area = calcularArea(5.0f, 3.0f);
  • 5.0f alimenta o parâmetro largura;
  • 3.0f alimenta o parâmetro altura;
  • a chamada produz o valor calculado pela função.

Como a informação chega ao corpo

1

Preparar valores

O programa possui literais, variáveis ou expressões que serão enviados.

largura = 5.0f
2

Fazer a chamada

O nome da função seguido dos parênteses inicia a execução do módulo.

calcularArea(...)
3

Associar posições

O primeiro argumento alimenta o primeiro parâmetro, e assim por diante.

5.0f → largura
4

Criar valores locais

Os parâmetros ficam disponíveis dentro da função com os valores recebidos.

altura = 3.0f
5

Executar e retornar

O corpo usa os parâmetros e devolve o resultado ao ponto da chamada.

return 15.0f

Rastreamento de uma chamada

rastrear_chamada.c
#include <stdio.h>

int somar(int a, int b) {
    int total = a + b;
    return total;
}

int main() {
    int x = 4;
    int y = 6;
    int resultado;

    resultado = somar(x, y);
    printf("%d\n", resultado);

    return 0;
}
1. Antes da chamada

O main possui x = 4 e y = 6.

Argumentos preparados: x e y.
2. Associação

Na chamada somar(x, y), a ordem define a correspondência.

x → a e y → b.
3. Dentro da função

Os parâmetros locais recebem cópias dos valores.

a = 4, b = 6 e total = 10.
4. Retorno ao ponto da chamada

O valor devolvido é armazenado em resultado.

resultado = 10. As variáveis x e y permanecem 4 e 6.

Exemplos progressivos

Calcular o triplo de um número

Uma chamada simples envia um valor literal para um parâmetro inteiro.

  • valor é o parâmetro
  • 5 é o argumento
  • triplo(5) é a chamada
  • o retorno produzido é 15
Nível básico
triplo.c
#include <stdio.h>

int triplo(int valor) {
    return valor * 3;
}

int main() {
    int resultado = triplo(5);
    printf("%d\n", resultado);
    return 0;
}

Calcular a média de duas notas

As variáveis do main fornecem os valores para dois parâmetros.

  • n1 e n2 são parâmetros locais
  • notaA e notaB são argumentos
  • os nomes podem ser diferentes
  • a posição mantém a correspondência
Nível intermediário
media.c
#include <stdio.h>

float calcularMedia(float n1, float n2) {
    return (n1 + n2) / 2.0f;
}

int main() {
    float notaA, notaB, media;

    scanf("%f %f", &notaA, &notaB);
    media = calcularMedia(notaA, notaB);

    printf("Media: %.2f\n", media);
    return 0;
}

Calcular o valor total com desconto

Três parâmetros com papéis e tipos distintos exigem atenção à ordem.

  • preco recebe o primeiro argumento
  • quantidade recebe o segundo
  • desconto recebe o terceiro
  • trocar as posições altera o significado da chamada
Nível intermediário
total_desconto.c
#include <stdio.h>

float calcularTotal(float preco, int quantidade, float desconto) {
    float subtotal = preco * quantidade;
    return subtotal - (subtotal * desconto / 100.0f);
}

int main() {
    float preco = 20.0f;
    int qtd = 3;
    float percentual = 10.0f;
    float total;

    total = calcularTotal(preco, qtd, percentual);
    printf("Total: %.2f\n", total);

    return 0;
}

Erros comuns

Chamar parâmetro de argumento

Por que acontece: os dois representam dados relacionados à função.

Como evitar: parâmetro aparece na definição; argumento aparece na chamada.

Enviar argumentos na ordem errada

Por que acontece: o aluno observa apenas os tipos e ignora o significado de cada posição.

Como evitar: compare cada argumento com o parâmetro correspondente antes de executar.

Esperar que o parâmetro altere a variável original

Por que acontece: o parâmetro pode ter recebido o valor de uma variável do main.

Como evitar: para tipos simples, lembre que a função trabalha com uma cópia local do valor.

Perguntas rápidas

Onde aparece um parâmetro?

Na definição ou declaração da função, acompanhado de seu tipo, como int numero.

Um argumento precisa ser sempre um número escrito diretamente?

Não. Pode ser um literal, uma variável ou uma expressão cujo resultado seja compatível com o parâmetro.

Os nomes do argumento e do parâmetro precisam ser iguais?

Não. A associação ocorre pela posição e pela compatibilidade de tipos, não pelo nome usado em cada contexto.
