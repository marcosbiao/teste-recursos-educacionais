Ideia central

Um laço termina quando seu estado caminha para uma condição falsa

Não basta escrever uma condição. Alguma variável usada no teste precisa mudar de forma previsível até que a repetição deixe de ser necessária.

Regra: estado inicial + teste + progresso = parada possível
1

Defina o começo

A variável de controle precisa receber um valor conhecido antes do primeiro teste.

2

Garanta progresso

Cada repetição deve aproximar o estado da condição que encerra o laço.

3

Preveja a saída

É preciso explicar qual valor ou evento fará o teste se tornar falso.

Inicialização: de onde o laço começa?

A inicialização estabelece o primeiro estado da variável de controle. Sem um valor inicial conhecido, não é possível prever o primeiro teste nem quantas vezes o laço será executado.

No mesmo exemplo:int i = 1;

O valor 1 corresponde ao primeiro número que deve ser exibido.

×

Teste: ainda é necessário repetir?

O teste deve expressar a regra de permanência no laço. Enquanto ele for verdadeiro, o bloco será executado novamente.

No mesmo exemplo:i <= 5

O laço continua enquanto o número atual ainda pertence ao intervalo de 1 a 5.

×

Atualização: o que muda a cada volta?

A atualização precisa modificar uma variável que participa da condição. Além disso, a direção da mudança deve aproximar o estado da saída.

No mesmo exemplo:i++;

Depois de imprimir um número, i aumenta uma unidade: 1, 2, 3, 4, 5 e 6.

×

Parada segura: quando o ciclo deixa de continuar?

Uma parada é segura quando pode ser prevista a partir da inicialização e das atualizações. Também pode incluir um limite adicional para impedir repetições indefinidas diante de entradas inesperadas.

No mesmo exemplo:quando i recebe 6, i <= 5 é falso

O sexto teste encerra o laço antes que o bloco seja executado novamente.

Três perguntas de segurança

1
O estado inicial é conhecido?

Sem inicialização, o primeiro teste pode ser imprevisível.

2
A variável do teste muda?

Se nada relevante for atualizado, o resultado da condição pode nunca mudar.

3
A mudança aponta para a saída?

Incrementar quando seria necessário decrementar pode afastar o estado da parada.

Exemplos progressivos

Nível básico

Exibir os números de 1 a 5

A quantidade de repetições é conhecida antes do início.

  • Inicialização: i = 1
  • Teste: i <= 5
  • Atualização: i++
  • Estado de saída: i = 6
O laço termina porque i aumenta em todas as repetições e inevitavelmente ultrapassa 5.
contagem.c
#include <stdio.h>

int main() {
    for (int i = 1; i <= 5; i++) {
        printf("%d\n", i);
    }

    return 0;
}
Nível intermediário

Somar números até o usuário digitar zero

A repetição depende de um valor sentinela informado durante a execução.

  • Inicialização: ler o primeiro número antes do laço
  • Teste: numero != 0
  • Atualização: ler um novo número dentro do laço
  • Estado de saída: numero = 0
A nova leitura é indispensável. Sem ela, o mesmo número seria testado para sempre.
sentinela.c
#include <stdio.h>

int main() {
    int numero, soma = 0;

    scanf("%d", &numero);

    while (numero != 0) {
        soma += numero;
        scanf("%d", &numero);
    }

    printf("Soma: %d\n", soma);
    return 0;
}
Nível intermediário

Validar uma senha com limite de tentativas

O laço possui duas saídas possíveis: acerto da senha ou uso das três tentativas.

  • Inicialização: erros = 0
  • Teste: senha incorreta e erros menores que 3
  • Atualização: incrementar erros após tentativa incorreta
  • Parada segura: acerto ou limite alcançado
O limite funciona como proteção adicional. Mesmo que a senha nunca seja correta, o programa não repete indefinidamente.
senha.c
#include <stdio.h>

int main() {
    const int SENHA = 1234;
    int tentativa, erros = 0;

    do {
        scanf("%d", &tentativa);

        if (tentativa != SENHA) {
            erros++;
        }
    } while (tentativa != SENHA && erros < 3);

    if (tentativa == SENHA)
        printf("Acesso permitido\n");
    else
        printf("Tentativas encerradas\n");

    return 0;
}

Erros comuns

Esquecer a atualização

Por que acontece: a condição depende de uma variável que nunca muda dentro do laço.

Como evitar: marque no código qual comando altera cada variável usada no teste.

Atualizar na direção errada

Por que acontece: o estado se afasta do valor que tornaria a condição falsa.

Como evitar: simule duas ou três repetições e verifique se a distância até a saída diminui.

Não renovar a entrada

Por que acontece: um laço com sentinela testa sempre o mesmo valor lido antes da repetição.

Como evitar: faça uma nova leitura em toda volta antes do próximo teste.

Perguntas rápidas

Uma condição correta garante, sozinha, que o laço termine?

Não. Também é necessário que alguma variável usada na condição seja atualizada de modo a tornar o teste falso em algum momento.

Por que i-- pode causar laço infinito em while (i <= 5) quando i começa em 1?

Porque i fica cada vez menor e continua satisfazendo a condição. A atualização afasta o estado da saída.

Qual é a função de um limite máximo de tentativas?

Ele cria uma segunda condição de encerramento e impede que o programa permaneça repetindo indefinidamente quando o evento esperado não ocorre.
