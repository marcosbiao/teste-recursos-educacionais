Ideia central

Uma consulta transforma uma pergunta em comparação

O programa recebe uma chave de busca ou um critério, percorre somente os registros cadastrados e compara o campo adequado. O resultado pode ser um registro, vários registros ou nenhum.

produtos[i].codigo == codigoProcurado compara a chave informada com o código do registro atual.

1

Escolha o campo

Defina se a consulta usa código, nome, posição, preço, nota ou outro campo.

2

Compare corretamente

Números usam operadores relacionais. Textos exigem funções como strcmp.

3

Trate o resultado

Mostre o registro encontrado, todos os aprovados no filtro ou uma mensagem de ausência.

Fluxo de uma consulta segura

1. Receber

Leia a chave ou o critério, como codigoProcurado.

2. Percorrer

Visite de 0 até quantidade - 1.

3. Comparar

Teste o campo do registro atual contra a pergunta.

4. Registrar

Exiba, conte ou guarde o índice quando houver correspondência.

5. Finalizar

Informe também quando nenhum registro for encontrado.

Rastreamento de uma busca por código

busca.c
int codigoProcurado = 205;
int posicao = -1;

for (int i = 0; i < quantidade; i++) {
    if (produtos[i].codigo == codigoProcurado) {
        posicao = i;
        break;
    }
}

Por que usar -1? Os índices válidos começam em 0. Assim, -1 pode representar “ainda não encontrado” sem confundir com uma posição real.

Quatro decisões antes de programar

1

Qual é a pergunta?

Um elemento específico ou todos os registros que atendem a uma regra?

2

Qual campo comparar?

Use código, nome, nota, preço ou outro campo que represente a pergunta.

3

Quando parar?

Uma chave única permite break. Um filtro exige percorrer tudo.

4

O que retornar?

O índice, o registro, a quantidade encontrada ou apenas a exibição dos resultados.

Exemplos progressivos

Básico

Mostrar o produto de uma posição informada

O usuário fornece a posição. Antes do acesso, o programa verifica se o índice pertence ao intervalo válido.

  • Acesso direto, sem busca sequencial.
  • Validação: posicao >= 0 e posicao < quantidade.
  • Uma posição inválida não deve ser acessada.
índicevalidaçãostruct

Consultar por posição só faz sentido quando o usuário conhece a organização interna do vetor.

consulta_posicao.c
int posicao;
scanf("%d", &posicao);

if (posicao >= 0 &&
    posicao < quantidade) {
    printf("%s - R$ %.2f\n",
           produtos[posicao].nome,
           produtos[posicao].preco);
} else {
    printf("Posicao invalida.\n");
}
Intermediário

Buscar produto por código e retornar o índice

A função percorre o cadastro e devolve a posição do produto. Se não encontrar, devolve -1.

  • O código funciona como chave de busca.
  • A função pode parar no primeiro resultado.
  • O chamador decide como apresentar o registro.
funçãoforbreak/return

Retornar o índice separa a lógica de busca da lógica de exibição.

busca_codigo.c
int buscarPorCodigo(Produto v[],
                      int n, int codigo) {
    for (int i = 0; i < n; i++) {
        if (v[i].codigo == codigo) {
            return i;
        }
    }
    return -1;
}

int p = buscarPorCodigo(produtos, quantidade, 205);
if (p != -1) {
    printf("Encontrado: %s\n", produtos[p].nome);
}
Intermediário avançado

Listar produtos baratos que ainda têm estoque

O filtro combina dois critérios e pode produzir vários resultados.

  • Preço de até R$ 50.
  • Estoque maior que zero.
  • Contagem para detectar ausência de resultados.
filtro&&contador

Não use break em um filtro quando todos os registros correspondentes devem ser mostrados.

filtro_produtos.c
int encontrados = 0;

for (int i = 0; i < quantidade; i++) {
    if (produtos[i].preco <= 50.0 &&
        produtos[i].estoque > 0) {
        printf("%s - R$ %.2f\n",
               produtos[i].nome,
               produtos[i].preco);
        encontrados++;
    }
}

if (encontrados == 0) {
    printf("Nenhum produto atende ao filtro.\n");
}

Erros comuns

Percorrer a capacidade máxima

Por que acontece: o aluno usa o tamanho total do vetor, mesmo com posições não preenchidas.

Como evitar: percorra somente até quantidade.

Comparar texto com ==

Por que acontece: a comparação numérica é aplicada diretamente a vetores de caracteres.

Como evitar: use strcmp para igualdade entre strings.

Parar no primeiro item do filtro

Por que acontece: um break é usado em uma consulta que deveria retornar vários registros.

Como evitar: diferencie busca única de filtro múltiplo.

Não tratar ausência

Por que acontece: o programa só possui saída para o caso de sucesso.

Como evitar: use índice -1, flag ou contador para detectar nenhum resultado.

Perguntas rápidas

Uma busca por código precisa percorrer o cadastro inteiro depois de encontrar a chave?

Não, quando o código é único. A busca pode retornar imediatamente ou usar break após guardar a posição.

Qual é a diferença entre retornar -1 e retornar 0?

O índice 0 é uma posição válida. O valor -1 fica fora do intervalo dos índices e pode representar “não encontrado”.

Por que um filtro normalmente não usa break?

Porque vários registros podem atender ao critério. O laço precisa continuar para localizar todos eles.

Resultado e ausência

O programa deve diferenciar sucesso, múltiplos resultados e nenhuma correspondência.

Busca única

Guarde a posição encontrada. Se continuar -1, o registro não existe.

Filtro

Conte quantos registros foram exibidos. Se a contagem for 0, o filtro não encontrou resultados.

Nunca use um campo de um registro antes de confirmar que a busca encontrou uma posição válida.
