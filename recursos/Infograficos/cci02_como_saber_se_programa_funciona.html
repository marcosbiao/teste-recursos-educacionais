Ideia central

Executar não é o mesmo que testar

Um programa pode compilar e ainda produzir uma resposta incorreta. Para testar de forma consciente, escolha uma entrada que você consegue calcular manualmente, defina o resultado esperado antes da execução e depois compare com a saída observada.

1

Escolha

Use valores pequenos, claros e fáceis de acompanhar.

2

Preveja

Calcule manualmente qual resposta deveria aparecer.

3

Compare

Verifique se resultado esperado e resultado observado coincidem.

Que valores devo escolher?

Valor pequeno comum

Verifica o comportamento mais esperado com uma entrada fácil de calcular.

Exemplo: 4

Valor neutro

O zero ajuda a revelar inicializações erradas, divisões indevidas e resultados acumulados.

Exemplo: 0

Valor no limite

Quando existe uma condição, teste exatamente o ponto em que a decisão muda.

Exemplo: média 7,0

Caminho contrastante

Teste valores que fazem o programa seguir respostas diferentes.

Exemplo: 6,9 e 7,0

Nível básico

Calcular o dobro de um número

O programa recebe um inteiro e mostra o valor multiplicado por 2.

Como não há decisão, use valores comuns, zero e um valor negativo.

dobro.c
#include <stdio.h>

int main() {
    int numero, dobro;

    scanf("%d", &numero);
    dobro = numero * 2;

    printf("Dobro: %d\n", dobro);
    return 0;
}
Nível intermediário

Informar aprovação pela média

O programa recebe a média. Valores maiores ou iguais a 7,0 devem produzir “Aprovado”; os demais, “Reprovado”.

Aqui o valor de limite é essencial, porque o operador usado é >=.

aprovacao.c
#include <stdio.h>

int main() {
    float media;

    scanf("%f", &media);

    if (media >= 7.0)
        printf("Aprovado\n");
    else
        printf("Reprovado\n");

    return 0;
}

#include <stdio.h>

int main() {
    int n, i, soma = 0;

    scanf("%d", &n);

    for (i = 1; i <= n; i++)
        soma = soma + i;

    printf("Soma: %d\n", soma);
    return 0;
}

Erros comuns

Executar sem prever

Por que acontece: o aluno observa qualquer saída e assume que ela está correta.

Como evitar: escreva o resultado esperado antes de rodar o programa.

Testar somente um valor

Por que acontece: um caso comum funciona, mas outros caminhos permanecem sem verificação.

Como evitar: inclua valor comum, limite e caminho contrastante quando houver decisão.

Alterar código e entrada ao mesmo tempo

Por que acontece: muitas mudanças dificultam saber o que corrigiu ou criou o erro.

Como evitar: repita os mesmos casos de teste depois de cada alteração.

Perguntas rápidas

Se o programa compilou, isso significa que o resultado está correto?

Não. A compilação indica que o código pôde ser traduzido, mas não garante que a lógica implementada corresponde ao problema.

Por que calcular a resposta antes de executar?

Porque o resultado esperado funciona como referência. Sem ele, não existe uma comparação objetiva para decidir se a saída está correta.

Um caso de teste que passou prova que o programa sempre funciona?

Não. Ele mostra apenas que o programa se comportou como esperado naquela entrada. Outros valores podem percorrer operações ou decisões diferentes.

Fechamento pedagógico

Depois de estudar este infográfico, o aluno deve conseguir planejar casos de teste simples, prever manualmente os resultados e comparar a saída observada com a resposta esperada para identificar possíveis falhas no programa.
