Ideia central

Planejar é definir responsabilidades antes de escrever detalhes

Um programa maior fica mais compreensível quando cada parte responde por uma tarefa específica. Em vez de concentrar leitura, cálculo, validação, armazenamento e saída no main, você cria blocos com papéis bem definidos.

O objetivo não é criar muitas funções. É criar partes úteis, com nomes que expliquem o que fazem e interfaces que deixem claro como os dados circulam.

Do enunciado ao plano

1. Marque as ações

Localize verbos como ler, calcular, validar, buscar e mostrar.

2. Agrupe por objetivo

Ações que trabalham para o mesmo resultado podem formar um único bloco.

3. Dê nomes claros

Prefira nomes que expressem ação, como calcularMedia ou buscarProduto.

4. Defina entradas e saídas

Decida quais dados o bloco recebe, modifica ou devolve para outra parte.

5. Monte a coordenação

Organize no main a ordem das chamadas e o fluxo dos resultados.

Exemplos progressivos

Nível básico

Converter Celsius para Fahrenheit

Mesmo um problema curto pode ser planejado em três responsabilidades para tornar o fluxo explícito.

  • Entrada: ler a temperatura em Celsius.
  • Processamento: aplicar a fórmula de conversão.
  • Saída: mostrar o resultado formatado.
lerCelsiusconvertermostrar
A divisão é didática. Em um programa real muito curto, algumas partes poderiam permanecer juntas. O importante é reconhecer os papéis.
conversao.c
#include <stdio.h>

float lerCelsius(void) {
    float c;
    scanf("%f", &c);
    return c;
}

float converter(float c) {
    return c * 1.8f + 32;
}

void mostrar(float f) {
    printf("Fahrenheit: %.1f\n", f);
}

int main(void) {
    float celsius = lerCelsius();
    float fahrenheit = converter(celsius);
    mostrar(fahrenheit);
    return 0;
}
Nível intermediário

Resumo de cinco notas

O vetor é preenchido por um bloco, processado por outros dois e apresentado por um procedimento.

  • lerNotas: cuida da entrada.
  • calcularMedia: devolve um resultado numérico.
  • contarAprovados: aplica uma regra.
  • exibirResumo: reúne a saída.
vetorforretornovoid
Observe que cada bloco usa apenas os dados de que precisa. A nota mínima é recebida como parâmetro, não escondida em uma variável global.
resumo_notas.c
#include <stdio.h>
#define N 5

void lerNotas(float notas[], int n) {
    for (int i = 0; i < n; i++) {
        do {
            scanf("%f", &notas[i]);
        } while (notas[i] < 0 || notas[i] > 10);
    }
}

float calcularMedia(const float notas[], int n) {
    float soma = 0;
    for (int i = 0; i < n; i++)
        soma += notas[i];
    return soma / n;
}

int contarAprovados(const float notas[], int n, float limite) {
    int total = 0;
    for (int i = 0; i < n; i++)
        if (notas[i] >= limite) total++;
    return total;
}

void exibirResumo(float media, int aprovados) {
    printf("Media: %.2f | Aprovados: %d\n", media, aprovados);
}

int main(void) {
    float notas[N];
    lerNotas(notas, N);
    float media = calcularMedia(notas, N);
    int aprovados = contarAprovados(notas, N, 7.0f);
    exibirResumo(media, aprovados);
    return 0;
}
Nível intermediário avançado

Cadastro de produtos com menu

O programa envolve várias operações. O planejamento separa o menu, o cadastro, a busca e a listagem.

  • exibirMenu: mostra opções e devolve a escolha.
  • cadastrarProduto: preenche um novo registro.
  • buscarPorCodigo: devolve a posição encontrada.
  • listarProdutos: percorre apenas os registros existentes.
structvetorswitchfunções
O exemplo mostra a arquitetura principal. Cada operação tem um contrato claro, e o main concentra apenas a coordenação do menu.
cadastro_modular.c
#include <stdio.h>
#define MAX 20

typedef struct {
    int codigo;
    float preco;
} Produto;

int exibirMenu(void) {
    int opcao;
    printf("1-Cadastrar 2-Buscar 3-Listar 0-Sair\n");
    scanf("%d", &opcao);
    return opcao;
}

void cadastrarProduto(Produto produtos[], int *quantidade) {
    if (*quantidade >= MAX) return;
    scanf("%d%f",
          &produtos[*quantidade].codigo,
          &produtos[*quantidade].preco);
    (*quantidade)++;
}

int buscarPorCodigo(const Produto produtos[], int n, int codigo) {
    for (int i = 0; i < n; i++)
        if (produtos[i].codigo == codigo) return i;
    return -1;
}

void listarProdutos(const Produto produtos[], int n) {
    for (int i = 0; i < n; i++)
        printf("%d - %.2f\n", produtos[i].codigo, produtos[i].preco);
}

int main(void) {
    Produto produtos[MAX];
    int quantidade = 0, opcao;

    do {
        opcao = exibirMenu();
        switch (opcao) {
            case 1: cadastrarProduto(produtos, &quantidade); break;
            case 2: {
                int codigo;
                scanf("%d", &codigo);
                int pos = buscarPorCodigo(produtos, quantidade, codigo);
                printf(pos >= 0 ? "Encontrado\n" : "Nao encontrado\n");
                break;
            }
            case 3: listarProdutos(produtos, quantidade); break;
        }
    } while (opcao != 0);

    return 0;
}

Erros comuns

Deixar tudo no main

Por que acontece: o código começa pequeno e cresce sem reorganização.

Como evitar: identifique responsabilidades independentes e extraia blocos com nomes claros.

Criar funções sem propósito

Por que acontece: cada linha ou comando vira uma função diferente.

Como evitar: agrupe ações que juntas produzem um resultado reconhecível.

Esconder dados em globais

Por que acontece: parece mais fácil acessar qualquer variável de qualquer lugar.

Como evitar: use parâmetros e retornos para tornar o fluxo de dados explícito.

Misturar responsabilidades

Por que acontece: uma função lê, calcula, altera dados e imprime ao mesmo tempo.

Como evitar: descreva a função em uma frase curta. Se precisar de vários verbos desconectados, revise a divisão.

Perguntas rápidas

Qual é o papel principal do main em um programa modular?

Coordenar a sequência geral: declarar os dados principais, chamar os blocos necessários, receber resultados e encerrar o programa. Os detalhes de cada tarefa ficam nas funções ou procedimentos correspondentes.

Uma função deve sempre executar apenas uma linha?

Não. Ela deve ter uma responsabilidade coerente. Essa responsabilidade pode exigir várias linhas, desde que todas contribuam para o mesmo objetivo.

Como saber quais dados uma função deve receber?

Liste apenas as informações necessárias para ela cumprir sua responsabilidade. Dados que não participam da tarefa não devem ser passados sem necessidade.

Coordenação

O bloco coordenador organiza a sequência geral e conecta as saídas de uma função às entradas da próxima. Em programas introdutórios, esse papel costuma ficar no main.
