Ideia central

O cadastro em memória desaparece quando o programa termina

Uma variável ou um vetor de structs existe apenas durante a execução. Para conservar os registros, o programa precisa convertê-los para um formato gravável, armazená-los em um arquivo e saber interpretar o mesmo formato ao recuperar os dados.

Ciclo da persistência: registro → gravar → fechar → abrir → ler → reconstruir ou consultar.

3

Recupere somente registros completos e trate o caso em que a consulta não encontra resultado.

Anatomia de uma gravação

salvar_produto.c
Legenda: Registro Abrir Gravar Fechar 
#include <stdio.h>
struct Produto {    int codigo;    char nome[40];    float preco;};
int main(void) {
    struct Produto produto = {17, "Caderno", 12.50};
    FILE *arquivo = fopen("produtos.txt", "a");
    if (arquivo == NULL) return 1;
    fprintf(arquivo, "%d %s %.2f\n",            produto.codigo, produto.nome, produto.preco);
    fclose(arquivo);
    return 0;
}

O contrato entre gravação e leitura

1. Mesma sequência

Ordem dos campos

Se a escrita usa codigo, nome, preco, a leitura deve recuperar codigo, nome, preco.

2. Tipos compatíveis

Formatadores

%d lê inteiro, %39s limita o texto e %f lê o preço.

3. Registro completo

Retorno da leitura

Exemplos progressivos

Salvar um aluno

Gravar um registro simples em uma linha de texto.

Nível básico
Registro: matrícula, nome sem espaços e média.
Modo: "a" para preservar os alunos anteriores.
Linha gerada: 2026 Ana 8.50.
salvar_aluno.c
#include <stdio.h>

struct Aluno {
    int matricula;
    char nome[40];
    float media;
};

int main(void) {
    struct Aluno aluno = {2026, "Ana", 8.5};
    FILE *arquivo = fopen("alunos.txt", "a");

    if (arquivo == NULL) return 1;

    fprintf(arquivo, "%d %s %.2f\n",
            aluno.matricula, aluno.nome, aluno.media);
    fclose(arquivo);
    return 0;
}

Listar produtos armazenados

Recuperar todos os registros completos até o fim do arquivo.

Nível intermediário
Modo: "r" para não alterar o cadastro.
Controle: o laço continua enquanto três campos forem lidos.
Saída: cada registro recuperado é mostrado na tela.
listar_produtos.c
#include <stdio.h>

struct Produto {
    int codigo;
    char nome[40];
    float preco;
};

int main(void) {
    struct Produto produto;
    FILE *arquivo = fopen("produtos.txt", "r");

    if (arquivo == NULL) return 1;

    while (fscanf(arquivo, "%d %39s %f",
                  &produto.codigo, produto.nome, &produto.preco) == 3) {
        printf("%d - %s - R$ %.2f\n",
               produto.codigo, produto.nome, produto.preco);
    }

    fclose(arquivo);
    return 0;
}

Buscar livro pelo código

Percorrer os registros até localizar uma chave específica.

Nível intermediário
Critério: comparar cada livro.codigo com o valor procurado.
Parada: usar break quando o registro for encontrado.
Ausência: informar quando o arquivo termina sem correspondência.
buscar_livro.c
#include <stdio.h>

struct Livro {
    int codigo;
    char titulo[50];
    int ano;
};

int main(void) {
    struct Livro livro;
    int procurado = 205, encontrado = 0;
    FILE *arquivo = fopen("livros.txt", "r");

    if (arquivo == NULL) return 1;

    while (fscanf(arquivo, "%d %49s %d",
                  &livro.codigo, livro.titulo, &livro.ano) == 3) {
        if (livro.codigo == procurado) {
            printf("%s, %d\n", livro.titulo, livro.ano);
            encontrado = 1;
            break;
        }
    }

    if (!encontrado) printf("Livro nao encontrado.\n");
    fclose(arquivo);
    return 0;
}

Erros comuns

Abrir com "w" ao cadastrar

Por que acontece: o aluno associa escrita apenas ao modo "w".

Como evitar: use "a" quando os registros anteriores devem permanecer.

Gravar e ler em ordens diferentes

Por que acontece: os formatadores não seguem o mesmo contrato.

Como evitar: documente a sequência dos campos e repita-a na leitura.

Usar registro incompleto

Por que acontece: o retorno de fscanf não é verificado.

Como evitar: processe apenas quando todos os campos esperados forem lidos.

Confundir memória com persistência

Por que acontece: o vetor de structs parece suficiente durante a execução.

Como evitar: grave alterações antes de encerrar e recupere os dados ao iniciar.

Perguntas rápidas

Por que um vetor de structs não mantém o cadastro depois que o programa termina?

Porque o vetor está na memória de execução. Quando o processo termina, essa memória deixa de representar o cadastro. O arquivo é o meio usado para persistir os registros.

Por que a escrita e a leitura devem usar a mesma ordem de campos?

Porque o arquivo não conhece o significado de cada valor. O programa interpreta a sequência segundo o formato definido. Se a ordem mudar, os dados serão atribuídos aos campos errados ou a leitura falhará.

Qual modo é mais adequado para adicionar um registro sem apagar os anteriores?

O modo "a", que grava no final do arquivo e preserva o conteúdo existente.
