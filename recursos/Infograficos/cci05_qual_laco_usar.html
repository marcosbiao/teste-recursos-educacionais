Ideia central

Os três laços repetem comandos, mas destacam controles diferentes

Escolher um laço não significa procurar o único que funciona. Em muitos problemas, mais de uma estrutura pode produzir o mesmo resultado. A melhor escolha é aquela que deixa explícito como a repetição começa, continua e termina.

Pergunta principal: o que controla o fim da repetição?
1

Quantidade conhecida

Quando o número de repetições já está definido, o for costuma deixar a contagem mais visível.

2

Condição variável

Quando a repetição depende de algo que pode ocorrer em momento incerto, o while costuma ser mais natural.

3

Execução obrigatória

Quando o bloco precisa ocorrer pelo menos uma vez antes do teste, o do while expressa essa regra diretamente.

Como decidir

1

A quantidade é conhecida?

Quando você sabe quantas vezes repetir, ou percorre um intervalo definido, considere for.

2

O teste vem antes?

Quando o bloco só pode executar se uma condição inicial for verdadeira, considere while.

3

Precisa executar uma vez?

Quando é necessário realizar a ação antes de verificar se haverá nova repetição, considere do while.

O mesmo resultado pode ser escrito de formas diferentes

Problema comum

Mostrar os números de 1 a 3. Os três laços podem resolver esse problema, mas o for comunica a contagem com maior concisão.

1

Inicializar: começar com i = 1.

2

Testar: continuar enquanto i <= 3.

3

Atualizar: aumentar i após cada exibição.

Com formais compacto
for (int i = 1; i <= 3; i++) {
    printf("%d ", i);
}
Com whilecontrole separado
int i = 1;
while (i <= 3) {
    printf("%d ", i);
    i++;
}
Com do whileprimeira execução garantida
int i = 1;
do {
    printf("%d ", i);
    i++;
} while (i <= 3);

Exemplos progressivos

Nível básico

Mostrar os números de 1 a 5

A quantidade de repetições é conhecida. O contador começa em 1, termina em 5 e aumenta de uma unidade.

Laço indicado: for
Controle: intervalo conhecido
Saída: 1 2 3 4 5
contagem.c
#include <stdio.h>

int main() {
    for (int i = 1; i <= 5; i++) {
        printf("%d ", i);
    }

    return 0;
}
Nível intermediário

Ler números até o usuário digitar zero

A quantidade de leituras não é conhecida. O valor zero funciona como sentinela e encerra a repetição.

Laço indicado: while
Controle: valor sentinela
Atenção: a variável precisa ser lida novamente dentro do laço
sentinela.c
#include <stdio.h>

int main() {
    int numero;

    scanf("%d", &numero);

    while (numero != 0) {
        printf("Lido: %d\n", numero);
        scanf("%d", &numero);
    }

    return 0;
}
Nível intermediário

Exibir um menu até escolher sair

O menu precisa aparecer pelo menos uma vez. Depois da escolha, a condição decide se o programa deve repetir.

Laço indicado: do while
Controle: opção escolhida
Detalhe sintático: existe ponto e vírgula após o while
menu.c
#include <stdio.h>

int main() {
    int opcao;

    do {
        printf("1 - Continuar\n0 - Sair\n");
        scanf("%d", &opcao);

        if (opcao == 1) {
            printf("Executando...\n");
        }
    } while (opcao != 0);

    return 0;
}

Erros comuns

Esquecer a atualização

Por que acontece: a variável usada na condição nunca muda.

Como evitar: identifique qual comando aproxima o estado da condição de parada.

Errar o limite da contagem

Por que acontece: confusão entre < e <=, causando uma repetição a menos ou a mais.

Como evitar: simule o primeiro e o último valor do contador.

Usar o laço apenas por hábito

Por que acontece: o aluno memoriza uma estrutura e ignora a forma de controle do problema.

Como evitar: verifique quantidade conhecida, posição do teste e necessidade de execução inicial.

Perguntas rápidas

Um laço for sempre pode ser substituído por while?

Em muitos casos, sim. A inicialização pode ser colocada antes do while e a atualização dentro do bloco. A escolha deve considerar qual escrita comunica melhor a lógica.

Qual laço garante que o bloco execute pelo menos uma vez?

O do while, porque a condição é testada somente depois da primeira execução do bloco.

Qual laço é mais indicado para percorrer 10 posições de um vetor?

Normalmente o for, pois a faixa de índices e a atualização do contador são conhecidas e ficam reunidas no cabeçalho.

Como funciona o for

×
Use quando: a inicialização, a condição e a atualização formam um controle regular, especialmente em contagens e percursos por índices.
Inicialização

Executa uma vez, antes do primeiro teste.

Condição

É verificada antes de cada repetição.

Atualização

Executa após cada passagem pelo bloco.

estrutura_for.c
// Exibe cinco valores
for (int i = 0; i < 5; i++) {
    printf("%d\n", i);
}
Teste antes do bloco

Como funciona o while

×
Use quando: a repetição depende de uma condição que pode mudar em momento incerto e o bloco só deve executar se essa condição já for verdadeira.
Teste inicial

A condição é verificada antes da primeira execução.

Execução zero

Se a condição começar falsa, o bloco não executa.

Responsabilidade

Algum comando do bloco deve alterar o estado relacionado à condição.

estrutura_while.c
int senha = 0;

while (senha != 1234) {
    printf("Digite a senha: ");
    scanf("%d", &senha);
}
Teste depois do bloco

Como funciona o do while

×
Use quando: o programa precisa executar uma ação antes de decidir se haverá outra repetição, como exibir um menu ou ler um valor antes de validá-lo.
Primeira execução

O bloco ocorre antes do primeiro teste.

Teste posterior

A condição decide se o fluxo retorna ao início.

Sintaxe

O comando termina com ponto e vírgula após while (condicao);.

estrutura_do_while.c
int nota;

do {
    printf("Nota entre 0 e 10: ");
    scanf("%d", &nota);
} while (nota < 0 || nota > 10);
