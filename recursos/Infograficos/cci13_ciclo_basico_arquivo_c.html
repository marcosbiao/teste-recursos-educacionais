Ideia central

O programa não acessa o arquivo diretamente pelo nome

fopen cria uma conexão entre o programa e o arquivo. Essa conexão é representada por um ponteiro do tipo FILE *. Depois de verificar a conexão, o programa pode ler ou escrever e, ao final, deve encerrá-la com fclose.

Sequência segura: abrir → verificar → usar → fechar.
1

Escolha o modo

O modo informa se o programa pretende ler, criar, substituir ou acrescentar dados.

2

Teste a abertura

Se fopen falhar, o resultado será NULL e o uso deve ser interrompido.

3

Encerre o acesso

fclose libera o recurso e conclui as operações pendentes de escrita.

Abrir com fopen

Nome do arquivo e modo de acesso

×
Primeiro argumento

Indica o caminho ou nome do arquivo, como "notas.txt".

Segundo argumento

Define a intenção. "r" lê, "w" cria ou substitui, e "a" acrescenta.

Mesmo exemplo: FILE *arquivo = fopen("notas.txt", "r"); tenta abrir notas.txt para leitura.

Verificar a abertura

Não use uma conexão que não existe

×
Quando falha

O arquivo pode não existir, o caminho pode estar incorreto ou o sistema pode negar o acesso.

Como reconhecer

Quando fopen falha, o ponteiro recebe NULL.

Mesmo exemplo: se arquivo == NULL, mostre uma mensagem e encerre o programa antes de chamar fscanf.

Ler ou escrever

A operação precisa combinar com o modo escolhido

×
Leitura

fscanf lê dados formatados. fgets lê uma linha de texto.

Escrita

fprintf grava dados formatados. fputs grava uma sequência de caracteres.

Mesmo exemplo: fscanf(arquivo, "%f", &nota); tenta ler uma nota do arquivo já aberto em modo "r".

Fechar com fclose

O ciclo termina quando o recurso é liberado

×
Por que fechar

Libera o recurso mantido pelo sistema e conclui operações de escrita que ainda estejam pendentes.

Depois de fechar

O ponteiro não deve voltar a ser usado sem uma nova chamada a fopen.

Mesmo exemplo: depois de terminar a leitura de notas.txt, execute fclose(arquivo);.

Três modos essenciais

"r"

Ler

Abre um arquivo existente para consulta. A abertura falha se o arquivo não existir.

Efeito: preserva o conteúdo.
"w"

Escrever do início

Cria um arquivo novo ou esvazia um arquivo que já existe antes de gravar.

Atenção: o conteúdo anterior é substituído.
"a"

Acrescentar

Cria o arquivo se necessário e grava novos dados no final do conteúdo existente.

Efeito: preserva e amplia o conteúdo.

Exemplos progressivos

Criar mensagem.txt

Gravar uma linha simples em um arquivo novo.

Nível básico
Modo: "w", pois o objetivo é criar ou substituir.
Escrita: fprintf envia o texto para o arquivo.
Resultado: o arquivo contém uma linha.
criar.c
#include <stdio.h>

int main(void) {
    FILE *arquivo = fopen("mensagem.txt", "w");

    if (arquivo == NULL) {
        return 1;
    }

    fprintf(arquivo, "Primeiro arquivo em C!\n");
    fclose(arquivo);

    return 0;
}

Ler notas até o fim

Somar valores existentes e calcular a média.

Nível intermediário
Modo: "r", pois o arquivo já deve existir.
Laço seguro: continua enquanto fscanf conseguir ler um valor.
Resultado: média calculada apenas com leituras válidas.
media.c
#include <stdio.h>

int main(void) {
    FILE *arquivo = fopen("notas.txt", "r");
    float nota, soma = 0;
    int quantidade = 0;

    if (arquivo == NULL) {
        return 1;
    }

    while (fscanf(arquivo, "%f", &nota) == 1) {
        soma += nota;
        quantidade++;
    }

    if (quantidade > 0) {
        printf("Media: %.2f\n", soma / quantidade);
    }

    fclose(arquivo);
    return 0;
}

Acrescentar produto ao cadastro

Preservar registros existentes e adicionar uma nova linha.

Nível intermediário
Modo: "a", pois os registros anteriores devem permanecer.
Formato: cada produto ocupa uma linha com código, nome e preço.
Resultado: a nova linha aparece no final do arquivo.
acrescentar.c
#include <stdio.h>

int main(void) {
    FILE *arquivo;
    int codigo;
    char nome[40];
    float preco;

    scanf("%d %39s %f", &codigo, nome, &preco);

    arquivo = fopen("produtos.txt", "a");
    if (arquivo == NULL) {
        return 1;
    }

    fprintf(arquivo, "%d %s %.2f\n",
            codigo, nome, preco);

    fclose(arquivo);
    return 0;
}

Erros comuns

Não verificar NULL

Por que acontece: o aluno presume que fopen sempre funciona.

Como evitar: teste o ponteiro imediatamente após a abertura.

Usar "w" sem perceber

Por que acontece: o modo é escolhido apenas porque permite escrever.

Como evitar: lembre que "w" apaga o conteúdo anterior.

Ler com feof como condição

Por que acontece: o fim do arquivo é testado antes da tentativa de leitura.

Como evitar: use o retorno de fscanf ou fgets para controlar o laço.

Esquecer fclose

Por que acontece: o programa parece terminar normalmente mesmo sem fechar.

Como evitar: associe cada abertura bem-sucedida a um fechamento.

Perguntas rápidas

Qual modo deve ser usado para consultar um arquivo que já existe sem alterar seu conteúdo?

Use "r", que abre o arquivo para leitura.

Por que o ponteiro retornado por fopen deve ser comparado com NULL?

Porque NULL indica que a abertura falhou. Nesse caso, funções de leitura ou escrita não devem ser chamadas com esse ponteiro.

Qual é a diferença principal entre os modos "w" e "a"?

"w" cria ou substitui o conteúdo do arquivo. "a" preserva o conteúdo e acrescenta novos dados no final.
