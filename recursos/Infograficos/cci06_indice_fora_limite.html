Ideia central

O índice precisa apontar para uma posição existente

Em C, a primeira posição tem índice 0. Por isso, uma estrutura com tamanho TAM termina no índice TAM - 1. Acessar antes de 0 ou a partir de TAM ultrapassa o espaço reservado.

0 <= índice && índice < tamanho

Antes de acessar, faça quatro verificações

1. Identifique a dimensão

Descubra o tamanho do vetor ou de cada dimensão da matriz.

2. Determine a faixa

Transforme o tamanho na faixa que começa em 0 e termina em tamanho menos 1.

3. Valide o índice

Confirme os limites inferior e superior antes de usar a posição.

4. Acesse a estrutura

Leia ou altere o elemento somente depois que a condição for verdadeira.

Exemplos progressivos

Exibir cinco valores

O laço visita exatamente os índices de 0 a 4.

  • Tamanho: 5
  • Último índice: 4
  • Condição segura: i < 5
Nível básico
vetor.c
#include <stdio.h>

#define TAM 5

int main() {
    int v[TAM] = {3, 6, 9, 12, 15};

    for (int i = 0; i < TAM; i++) {
        printf("%d ", v[i]);
    }

    return 0;
}

Percorrer uma matriz 2 por 3

Cada laço respeita o tamanho de sua dimensão.

  • Linhas válidas: 0 e 1
  • Colunas válidas: 0, 1 e 2
  • Condições: i < 2 e j < 3
Nível intermediário
matriz.c
#include <stdio.h>

int main() {
    int m[2][3] = {
        {4, 7, 2},
        {8, 5, 9}
    };

    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", m[i][j]);
        }
    }

    return 0;
}

Consultar uma posição escolhida

O acesso acontece somente após a validação completa.

  • Limite inferior: índice maior ou igual a 0
  • Limite superior: índice menor que TAM
  • Mensagem clara quando a posição for inválida
Nível intermediário
consulta.c
#include <stdio.h>

#define TAM 5

int main() {
    int v[TAM] = {11, 22, 33, 44, 55};
    int indice;

    scanf("%d", &indice);

    if (indice >= 0 && indice < TAM) {
        printf("Valor: %d\n", v[indice]);
    } else {
        printf("Indice invalido.\n");
    }

    return 0;
}

Erros comuns

Usar <= tamanho

Por que acontece: o aluno confunde quantidade de elementos com último índice.

Como evitar: use i < tamanho quando o índice começa em 0.

Validar apenas o limite superior

Por que acontece: o programa verifica se o índice é menor que o tamanho, mas aceita valores negativos.

Como evitar: teste os dois lados da faixa.

Trocar linha e coluna

Por que acontece: as dimensões da matriz são tratadas como se tivessem o mesmo tamanho.

Como evitar: associe o primeiro índice às linhas e o segundo às colunas.

Perguntas rápidas

Qual é o último índice válido de um vetor com 8 posições?

O último índice é 7, pois os índices começam em 0.

Por que i <= TAM costuma estar errado ao percorrer um vetor?

Porque a condição permite que i assuma o valor TAM, mas essa posição não existe. O último índice válido é TAM menos 1.

O que deve ser validado antes de acessar m[linha][coluna]?

A linha precisa estar entre 0 e o número de linhas menos 1. A coluna precisa estar entre 0 e o número de colunas menos 1.

A condição decide o último acesso

Uma repetição a mais já é suficiente para sair do vetor.

Compare as condições

i <= 5 permite 0, 1, 2, 3, 4 e 5. O índice 5 é inválido.
i < 5 permite somente 0, 1, 2, 3 e 4. Todos são válidos.
Ao iniciar em 0, prefira pensar: repetir enquanto o índice for menor que o tamanho.

A matriz possui dois limites

Cada índice precisa respeitar o tamanho de sua dimensão.

Exemplo: int m[2][3];

Linhas válidas: 0 e 1. Colunas válidas: 0, 1 e 2.

m[1][2] é válido. m[2][2] e m[1][3] são inválidos.
Use um limite para i e outro para j. Não suponha que linhas e colunas têm o mesmo tamanho.

Valide antes de acessar

Um índice digitado pode ser negativo ou maior que a estrutura.

Condição completa

indice >= 0 && indice < TAM

O acesso v[indice] deve ficar dentro do bloco executado apenas quando essa condição for verdadeira.

Não basta verificar indice < TAM. Valores negativos também estão fora do limite.
