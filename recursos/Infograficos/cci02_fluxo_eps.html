Ideia central

Todo programa transforma uma situação

O fluxo E P S ajuda a enxergar o programa como um caminho. A entrada fornece os dados, o processamento aplica operações ou regras e a saída comunica o resultado. Nem todo programa terá muitas instruções, mas essas três funções ajudam a organizar o raciocínio.

1

Receber

Identifique quais informações o programa precisa conhecer.

2

Transformar

Defina o cálculo, comparação ou decisão necessária.

3

Comunicar

Mostre somente a resposta necessária e em formato compreensível.

Fluxo E P S no mesmo exemplo

Entrada: os dados que iniciam o fluxo

No problema da média, o programa precisa conhecer as duas notas antes de realizar qualquer cálculo.

Como reconhecer a entrada

1

Procure no enunciado os valores que serão fornecidos.

2

Defina variáveis compatíveis com o tipo desses dados.

3

Use comandos de leitura para armazenar cada valor.

nota1nota2tipo float

Trecho correspondente em C

entrada.c
float nota1, nota2;

scanf("%f", &nota1);
scanf("%f", &nota2);

Processamento: a transformação dos dados

As duas notas são combinadas por uma fórmula. O resultado dessa operação é armazenado em uma nova variável.

Como reconhecer o processamento

1

Localize o verbo que expressa a ação: calcular, comparar, converter ou decidir.

2

Escreva a fórmula ou regra usando os dados de entrada.

3

Armazene o resultado quando ele for necessário em outra etapa.

somardividir por 2guardar em media

Trecho correspondente em C

processamento.c
float media;

media = (nota1 + nota2) / 2.0;

Saída: a resposta produzida pelo programa

Depois do cálculo, a média é apresentada com uma mensagem que identifica o significado do valor.

Como reconhecer a saída

1

Descubra qual resposta o enunciado pede para mostrar.

2

Escolha a variável que contém o resultado correto.

3

Inclua uma mensagem curta para tornar a saída compreensível.

resultado: mediaduas casas decimaismensagem clara

Trecho correspondente em C

saida.c
printf("Media: %.2f\n", media);

Rastreie o caminho dos dados

Um modo simples de verificar o programa é acompanhar um conjunto de valores do início ao fim.

Entradanota1 = 8,0 e nota2 = 6,0

Os valores são lidos e armazenados nas variáveis.

Processamentomedia = (8,0 + 6,0) / 2

O cálculo produz o valor 7,0.

SaídaMedia: 7.00

O resultado é exibido com uma mensagem identificadora.

Exemplos progressivos

Troque as abas para observar como o fluxo E P S permanece reconhecível em problemas com cálculo simples, decisão e combinação de operações.

Nível inicial

Converter Celsius para Fahrenheit

O programa recebe uma temperatura em Celsius, aplica a fórmula de conversão e mostra o valor em Fahrenheit.

EntradaTemperatura em Celsius
Processamentofahrenheit = celsius * 9 / 5 + 32
SaídaTemperatura convertida
temperatura.c
#include <stdio.h>

int main() {
    float celsius, fahrenheit;

    scanf("%f", &celsius);

    fahrenheit = celsius * 9.0 / 5.0 + 32.0;

    printf("Fahrenheit: %.1f\n", fahrenheit);
    return 0;
}
Nível intermediário

Calcular compra com desconto

O programa recebe o valor da compra. Se o valor for pelo menos 200, aplica 10% de desconto; caso contrário, mantém o total.

EntradaValor da compra
ProcessamentoComparar o valor e calcular o total com ou sem desconto
SaídaTotal a pagar
desconto.c
#include <stdio.h>

int main() {
    float compra, total;

    scanf("%f", &compra);

    if (compra >= 200.0)
        total = compra * 0.90;
    else
        total = compra;

    printf("Total: R$ %.2f\n", total);
    return 0;
}
Nível intermediário

Calcular média e informar situação

O programa lê três notas, calcula a média e informa se o estudante foi aprovado ou reprovado.

EntradaTrês notas
ProcessamentoCalcular a média e compará-la com 7,0
SaídaMédia e situação do estudante
situacao.c
#include <stdio.h>

int main() {
    float n1, n2, n3, media;

    scanf("%f %f %f", &n1, &n2, &n3);
    media = (n1 + n2 + n3) / 3.0;

    printf("Media: %.2f\n", media);
    if (media >= 7.0)
        printf("Aprovado\n");
    else
        printf("Reprovado\n");

    return 0;
}

Erros comuns

Ler um dado e não utilizá-lo

Por que acontece: a entrada é criada sem relação com o cálculo ou a decisão.

Como evitar: confira onde cada variável lida participa do processamento.

Mostrar a variável errada

Por que acontece: o programa calcula um resultado, mas imprime um valor de entrada.

Como evitar: identifique qual variável contém a resposta final antes do printf.

Confundir fórmula com saída

Por que acontece: o aluno tenta exibir dados antes de concluir as operações necessárias.

Como evitar: mantenha clara a ordem: receber, processar e somente depois comunicar.

Perguntas rápidas

Uma variável declarada é sempre uma entrada?

Não. Uma variável pode guardar um dado recebido, um resultado intermediário ou a resposta final. Ela se torna entrada quando recebe um dado vindo de fora do processamento.

Uma estrutura if pertence a qual etapa do fluxo?

Ao processamento, porque aplica uma regra e decide qual caminho será seguido com base nos dados disponíveis.

O programa precisa sempre usar scanf para ter entrada?

Não. Dados também podem vir de arquivos, sensores, parâmetros ou valores previamente definidos. Em programas introdutórios, scanf é uma forma frequente de entrada pelo teclado.
