Ideia central

Enxergue o caminho da informação

Um enunciado costuma misturar dados, ações e resultados no mesmo texto. Para torná-lo programável, reorganize essas informações em três perguntas: o que o programa precisa receber, o que precisa fazer e o que deve informar ao final.

Entrada, processamento e saída representam o raciocínio da solução, não apenas comandos de código.
Leia o enunciado

Procure pistas no texto.

Entrada

Quais dados o programa ainda não conhece e precisa receber?

Cálculo

O que precisa ser somado, convertido, contado ou transformado?

Decisão

Existe alguma condição que altera o resultado ou o caminho seguido?

Saída

Qual resposta precisa ser mostrada de forma clara ao usuário?

Do texto ao esquema do programa

Movimento 1

Marque os verbos

Palavras como ler, calcular, comparar e mostrar indicam ações importantes.

Movimento 2

Localize os dados

Identifique os valores fornecidos pelo usuário e as constantes já definidas.

Movimento 3

Classifique

Separe cada elemento em entrada, processamento ou saída.

Movimento 4

Conecte

Verifique quais entradas alimentam cada cálculo ou decisão.

Movimento 5

Valide a resposta

Confirme se a saída pedida pode ser produzida com os dados identificados.

Faça a pergunta de controle: consigo explicar de onde vem cada valor mostrado na saída?

Exemplos por nível de complexidade

Nível inicial

Calcular a área de um retângulo

O programa recebe a base e a altura, calcula a área e mostra o resultado.

Saídaárea
ParteAplicação
EntradaBase e altura do retângulo
ProcessamentoMultiplicar base pela altura
SaídaValor calculado da área

Código curto em C

Fluxo linear simples
#include <stdio.h>

int main() {
    float base, altura, area;

    scanf("%f %f", &base, &altura);
    area = base * altura;

    printf("Area: %.2f\n", area);
    return 0;
}
Nível intermediário

Converter segundos em horas, minutos e segundos

O programa recebe uma duração total em segundos e a reorganiza em três unidades.

Entradatotal em segundos
→
Processamentodivisão e resto
→
Saídah, min e s
ParteAplicação
EntradaTotal de segundos
Processamento 1Calcular horas com divisão por 3600
Processamento 2Usar o resto para calcular minutos e segundos
SaídaDuração separada em três unidades

Código curto em C

Uma entrada, várias transformações
#include <stdio.h>

int main() {
    int total, horas, minutos, segundos;
    int restante;

    scanf("%d", &total);

    horas = total / 3600;
    restante = total % 3600;
    minutos = restante / 60;
    segundos = restante % 60;

    printf("%d h %d min %d s\n",
           horas, minutos, segundos);

    return 0;
}
Nível intermediário

Calcular o total de uma compra com desconto

O programa recebe o valor da compra, aplica 10% de desconto quando o valor é igual ou superior a 200 e mostra o total final.

Entradavalor da compra
→
Decisãovalor ≥ 200?
→
Saídadesconto e total
ParteAplicação
EntradaValor original da compra
ProcessamentoCalcular 10% quando a condição for atendida
DecisãoComparar o valor da compra com 200
SaídaDesconto aplicado e total final

Código curto em C

Processamento com decisão
#include <stdio.h>

int main() {
    float compra, desconto, total;

    scanf("%f", &compra);

    if (compra >= 200)
        desconto = compra * 0.10;
    else
        desconto = 0;

    total = compra - desconto;

    printf("Desconto: %.2f\n", desconto);
    printf("Total: %.2f\n", total);

    return 0;
}

Erros comuns

Tratar toda informação como entrada

Por que acontece: o aluno confunde dados fornecidos pelo usuário com valores fixos já definidos no enunciado.

Como evitar: pergunte quais valores realmente precisam ser lidos durante a execução.

Confundir processamento com saída

Por que acontece: expressões como “mostrar a média” escondem que a média precisa ser calculada antes.

Como evitar: separe a ação que produz o resultado da ação que o apresenta.

Ignorar uma regra de decisão

Por que acontece: o aluno identifica o cálculo, mas não percebe palavras como se, caso ou quando.

Como evitar: destaque condições que podem alterar o processamento ou a saída.

Perguntas rápidas

Toda informação numérica presente no enunciado é uma entrada?

Não. Alguns valores são constantes já definidas pelo problema, como uma taxa fixa, um limite ou o número 60 em uma conversão de tempo.

O processamento pode conter uma decisão?

Sim. Processamento inclui cálculos, transformações e decisões necessárias para produzir a resposta.

Como verificar se a separação foi feita corretamente?

Confirme se todas as saídas podem ser explicadas a partir das entradas e das operações ou regras identificadas.

Fechamento pedagógico

Depois de estudar este infográfico, o aluno deve conseguir ler um enunciado, identificar entradas, representar cálculos e decisões como processamento e definir com precisão a saída esperada antes de programar em C.
