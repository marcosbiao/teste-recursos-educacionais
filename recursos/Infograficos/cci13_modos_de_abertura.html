Ideia central

O modo de abertura é uma decisão sobre os dados

O segundo argumento de fopen não serve apenas para informar uma operação técnica. Ele define se o programa deve preservar, substituir ou acrescentar conteúdo.

Antes de escrever fopen("dados.txt", "?"), responda: quero apenas ler, recomeçar o arquivo ou adicionar sem perder o que já existe?

Modo de substituição

"w": começar um conteúdo novo

O modo "w" abre o arquivo para escrita. Se o arquivo já existir, seu conteúdo é truncado, isto é, reduzido a zero bytes antes da primeira gravação.

Exemplo: ao abrir dados.txt com "w" e gravar Carla, os nomes Ana e Bruno deixam de existir no arquivo.

Modo de acréscimo

"a": adicionar sem apagar

O modo "a" abre o arquivo para escrita no final. Se ele não existir, será criado. Se já existir, o conteúdo anterior permanece.

Modos combinados

+: leitura e escrita no mesmo arquivo

O sinal + acrescenta permissão de leitura e escrita, mas não elimina o comportamento do modo principal.

Regra: r+ exige arquivo existente e preserva os dados; w+ cria ou apaga o conteúdo; a+ cria ou preserva e mantém a escrita no final.
r+Preserva e começa no início
w+Apaga e recomeça
a+Preserva e escreve no final
Atenção: usar + não torna o modo automaticamente seguro. w+ continua apagando o conteúdo existente.

Escolha o modo pela intenção

Apenas consultar?

Os dados devem permanecer exatamente como estão.

use "r"

Gerar tudo novamente?

O conteúdo antigo não deve fazer parte do novo arquivo.

use "w"

Adicionar um registro?

O novo dado deve entrar depois do conteúdo já salvo.

use "a"

Precisa ler e alterar?

Escolha um modo com + e confirme seu efeito principal.

r+ · w+ · a+

Três verificações antes de usar "w"

O conteúdo antigo pode ser descartado?

Use "w" somente quando a substituição completa fizer parte do requisito.

O arquivo é fonte ou resultado?

Arquivos de entrada normalmente usam "r". Relatórios recriados podem usar "w".

É cadastro ou histórico?

Quando registros anteriores precisam permanecer, "a" é mais adequado que "w".

Abertura segura em C

adicionar_nome.c
#include <stdio.h> int main(void) { FILE *arquivo; arquivo = fopen("dados.txt", "a"); if (arquivo == NULL) { printf("Erro ao abrir o arquivo.\n"); return 1; } fprintf(arquivo, "Carla\n"); fclose(arquivo); return 0; }
1

Modo coerente

"a" foi escolhido porque o programa acrescentará Carla sem apagar Ana e Bruno.

2

Falha verificada

O retorno de fopen é testado antes de qualquer operação de escrita.

3

Arquivo fechado

fclose encerra o uso e permite que os dados pendentes sejam enviados ao arquivo.

Erros comuns

Usar "w" para adicionar

Por que acontece: o aluno associa escrita apenas ao modo "w".

Como evitar: use "a" quando o conteúdo anterior precisa permanecer.

Achar que o conteúdo só é apagado no fprintf

Por que acontece: o efeito da abertura é confundido com o efeito da gravação.

Como evitar: lembre que fopen(..., "w") já trunca o arquivo existente.

Ignorar a falha de "r"

Por que acontece: o programa assume que o arquivo sempre existe.

Como evitar: teste se o ponteiro retornado por fopen é NULL.

Perguntas rápidas

Qual modo adiciona um registro sem apagar os registros anteriores?

O modo "a". Ele preserva o conteúdo existente e posiciona as escritas no final do arquivo.

O que acontece ao abrir um arquivo existente com "w" e fechá-lo sem escrever nada?

O arquivo permanece existente, mas seu conteúdo anterior é apagado, pois o truncamento ocorre durante a abertura.

Adicionar + torna qualquer modo seguro contra perda de dados?

Não. O modo "w+" continua apagando o conteúdo existente. O + apenas permite leitura e escrita.
